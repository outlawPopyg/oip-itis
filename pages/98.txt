					<div id="contentSub"><div id="mw-content-subtitle"></div></div>
					
					
					<div id="mw-content-text" class="mw-body-content"><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Object-oriented software design pattern</div>
<p>In <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a>, the <b>factory method pattern</b> is a <a href="/wiki/Software_design_pattern" title="Software design pattern">design pattern</a> that uses factory methods to deal with the problem of <a href="/wiki/Object_creation" class="mw-redirect" title="Object creation">creating objects</a> without having to specify their exact <a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">classes</a>. Rather than by calling a <a href="/wiki/Constructor_(object-oriented_programming)" title="Constructor (object-oriented programming)">constructor</a>, this is accomplished by invoking a factory method to create an object. Factory methods can be specified in an <a href="/wiki/Interface_(object-oriented_programming)" title="Interface (object-oriented programming)">interface</a> and implemented by subclasses or implemented in a base class and optionally <a href="/wiki/Method_overriding" title="Method overriding">overridden</a> by subclasses. It is one of the 23 classic design patterns described in the book <i><a href="/wiki/Design_Patterns" title="Design Patterns">Design Patterns</a></i> (often referred to as the "Gang of Four" or simply "GoF") and is subcategorized as a <a href="/wiki/Creational_pattern" title="Creational pattern">creational pattern</a>.<sup id="cite_ref-FOOTNOTEGammaHelmJohnsonVlissides1995107_1-0" class="reference"><a href="#cite_note-FOOTNOTEGammaHelmJohnsonVlissides1995107-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup>
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Overview">Overview</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=1" title="Edit section: Overview"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The factory method design pattern solves problems such as:
</p>
<ul><li>How can an object's <a href="/wiki/Subclass_(computer_science)" class="mw-redirect" title="Subclass (computer science)">subclasses</a> redefine its subsequent and distinct implementation? The pattern involves creation of a factory method within the <a href="/wiki/Superclass_(computer_science)" class="mw-redirect" title="Superclass (computer science)">superclass</a> that defers the object's creation to a subclass's factory method.</li>
<li>How can an object's instantiation be deferred to a subclass? Create an object by calling a factory method instead of directly calling a constructor.</li></ul>
<p>This enables the creation of subclasses that can change the way in which an object is created (for example, by redefining which class to instantiate).
</p>
<div class="mw-heading mw-heading2"><h2 id="Definition">Definition</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=2" title="Edit section: Definition"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>According to <i><a href="/wiki/Design_Patterns" title="Design Patterns">Design Patterns: Elements of Reusable Object-Oriented Software</a></i>: "Define an interface for creating an object, but let subclasses decide which class to instantiate. Factory method lets a class defer instantiation to subclasses."<sup id="cite_ref-gof_2-0" class="reference"><a href="#cite_note-gof-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Creating an object often requires complex processes not appropriate to include within a composing object. The object's creation may lead to a significant duplication of code, may require information inaccessible to the composing object, may not provide a sufficient level of abstraction or may otherwise not be included in the composing object's <a href="/wiki/Concern_(computer_science)" title="Concern (computer science)">concerns</a>. The factory method design pattern handles these problems by defining a separate <a href="/wiki/Method_(computer_science)" class="mw-redirect" title="Method (computer science)">method</a> for creating the objects, which subclasses can then override to specify the <a href="/wiki/Subtyping" title="Subtyping">derived type</a> of product that will be created.
</p><p>The factory method pattern relies on inheritance, as object creation is delegated to subclasses that implement the factory method to create objects.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup>
The pattern can also rely on the implementation of an <a href="/wiki/Interface_(object-oriented_programming)" title="Interface (object-oriented programming)">interface</a>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Structure">Structure</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=3" title="Edit section: Structure"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-heading mw-heading3"><h3 id="UML_class_diagram">UML class diagram</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=4" title="Edit section: UML class diagram"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<figure class="mw-halign-none" typeof="mw:File/Frame"><a href="/wiki/File:W3sDesign_Factory_Method_Design_Pattern_UML.jpg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/4/43/W3sDesign_Factory_Method_Design_Pattern_UML.jpg" decoding="async" width="500" height="240" class="mw-file-element" data-file-width="500" data-file-height="240" /></a><figcaption>A sample UML class diagram for the Factory Method design pattern. <sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup></figcaption></figure>
<p>In the above <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">UML</a> <a href="/wiki/Class_diagram" title="Class diagram">class diagram</a>, the <code>Creator</code> class that requires a <code>Product</code> object does not instantiate the <code>Product1</code> class directly. Instead, the <code>Creator</code> refers to a separate <code>factoryMethod()</code> to create a product object, which makes the <code>Creator</code> independent of the exact concrete class that is instantiated. Subclasses of <code>Creator</code> can redefine which class to instantiate. In this example, the <code>Creator1</code> subclass implements the abstract <code>factoryMethod()</code> by instantiating the <code>Product1</code> class.
</p>

</p>

<p>The program output is like
</p>
<div class="mw-highlight mw-highlight-lang-c++ mw-content-ltr" dir="ltr"><pre><span></span><span class="k">this</span><span class="o">=</span><span class="mh">0x6e5e90</span><span class="w"> </span><span class="n">print</span><span class="w"> </span><span class="n">MINE</span>
<span class="k">this</span><span class="o">=</span><span class="mh">0x6e62c0</span><span class="w"> </span><span class="n">print</span><span class="w"> </span><span class="n">YOURS</span>
</pre></div>
<p>A maze game may be played in two modes, one with regular rooms that are only connected with adjacent rooms, and one with magic rooms that allow players to be transported at random. 
</p>
<div class="mw-heading mw-heading3"><h3 id="Structure_2">Structure</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=6" title="Edit section: Structure"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<figure class="mw-halign-center" typeof="mw:File"><a href="/wiki/File:New_WikiFactoryMethod.png" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/d/df/New_WikiFactoryMethod.png/734px-New_WikiFactoryMethod.png" decoding="async" width="734" height="399" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/d/df/New_WikiFactoryMethod.png 1.5x" data-file-width="910" data-file-height="495" /></a><figcaption></figcaption></figure>
<p><code>Room</code> is the base class for a final product (<code>MagicRoom</code> or <code>OrdinaryRoom</code>). <code>MazeGame</code> declares the abstract factory method to produce such a base product. <code>MagicRoom</code> and <code>OrdinaryRoom</code> are subclasses of the base product implementing the final product. <code>MagicMazeGame</code> and <code>OrdinaryMazeGame</code> are subclasses of <code>MazeGame</code> implementing the factory method producing the final products. Factory methods thus decouple callers (<code>MazeGame</code>) from the implementation of the concrete classes. This makes the <code>new</code> operator redundant, allows adherence to the <a href="/wiki/Open%E2%80%93closed_principle" title="Open–closed principle">open–closed principle</a> and makes the final product more flexible in the event of change.
</p>
<div class="mw-heading mw-heading3"><h3 id="Example_implementations">Example implementations</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=7" title="Edit section: Example implementations"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-heading mw-heading4"><h4 id="C#"><span id="C.23"></span><a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a></h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=8" title="Edit section: C#"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-highlight mw-highlight-lang-csharp mw-content-ltr" dir="ltr"><pre><span></span><span class="c1">// Empty vocabulary of actual object</span>
<span class="k">public</span><span class="w"> </span><span class="k">interface</span><span class="w"> </span><span class="n">IPerson</span>
<span class="p">{</span>
<span class="w">    </span><span class="kt">string</span><span class="w"> </span><span class="nf">GetName</span><span class="p">();</span>
<span class="p">}</span>

<span class="k">public</span><span class="w"> </span><span class="k">class</span><span class="w"> </span><span class="nc">Villager</span><span class="w"> </span><span class="p">:</span><span class="w"> </span><span class="n">IPerson</span>

<span class="w">    </span><span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span><span class="w"> </span><span class="k">class</span><span class="w"> </span><span class="nc">CityPerson</span><span class="w"> </span><span class="p">:</span><span class="w"> </span><span class="n">IPerson</span>
<span class="p">{</span>
<span class="w">    </span><span class="k">public</span><span class="w"> </span><span class="kt">string</span><span class="w"> </span><span class="nf">GetName</span><span class="p">()</span>
<span class="w">    </span><span class="p">{</span>
<span class="w">        </span><span class="k">return</span><span class="w"> </span><span class="s">&quot;City Person&quot;</span><span class="p">;</span>
<span class="w">    </span><span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span><span class="w"> </span><span class="k">enum</span><span class="w"> </span><span class="n">PersonType</span>

<span class="c1">/// &lt;summary&gt;</span>
<span class="c1">/// Implementation of Factory - Used to create objects.</span>
<span class="c1">/// &lt;/summary&gt;</span>
<span class="k">public</span><span class="w"> </span><span class="k">class</span><span class="w"> </span><span class="nc">PersonFactory</span>
<span class="p">{</span>
<span class="w">    </span><span class="k">public</span><span class="w"> </span><span class="n">IPerson</span><span class="w"> </span><span class="nf">GetPerson</span><span class="p">(</span><span class="n">PersonType</span><span class="w"> </span><span class="n">type</span><span class="p">)</span>
<span class="w">    </span><span class="p">{</span>
<span class="w">        </span><span class="k">switch</span><span class="w"> </span><span class="p">(</span><span class="n">type</span><span class="p">)</span>
<span class="w">        </span><span class="p">{</span>
<span class="w">            </span><span class="k">case</span><span class="w"> </span><span class="n">PersonType</span><span class="p">.</span><span class="n">Rural</span><span class="p">:</span>
<span class="w">                </span><span class="k">return</span><span class="w"> </span><span class="k">new</span><span class="w"> </span><span class="nf">Villager</span><span class="p">();</span>
<span class="w">            </span><span class="k">case</span><span class="w"> </span><span class="n">PersonType</span><span class="p">.</span><span class="n">Urban</span><span class="p">:</span>
<span class="w">                </span><span class="k">return</span><span class="w"> </span><span class="k">new</span><span class="w"> </span><span class="nf">CityPerson</span><span class="p">();</span>
<span class="w">            </span><span class="k">default</span><span class="p">:</span>
<span class="w">                </span><span class="k">throw</span><span class="w"> </span><span class="k">new</span><span class="w"> </span><span class="nf">NotSupportedException</span><span class="p">();</span>

<p>The above code depicts the creation of an interface called <code>IPerson</code> and two implementations called <code>Villager</code> and <code>CityPerson</code>. Based on the type passed to the <code>PersonFactory</code> object, the original concrete object is returned as the interface <code>IPerson</code>.
</p><p>A factory method is just an addition to the <code>PersonFactory</code> class. It creates the object of the class through interfaces but also allows the subclass to decide which class is instantiated.
</p>
<div class="mw-highlight mw-highlight-lang-csharp mw-content-ltr" dir="ltr"><pre><span></span><span class="k">public</span><span class="w"> </span><span class="k">interface</span><span class="w"> </span><span class="n">IProduct</span>
<span class="p">{</span>
<span class="w">    </span><span class="kt">string</span><span class="w"> </span><span class="nf">GetName</span><span class="p">();</span>
<span class="w">    </span><span class="kt">bool</span><span class="w"> </span><span class="nf">SetPrice</span><span class="p">(</span><span class="kt">double</span><span class="w"> </span><span class="n">price</span><span class="p">);</span>
<span class="p">}</span>

<span class="k">public</span><span class="w"> </span><span class="k">class</span><span class="w"> </span><span class="nc">Phone</span><span class="w"> </span><span class="p">:</span><span class="w"> </span><span class="n">IProduct</span>
<span class="p">{</span>
<span class="w">    </span><span class="k">private</span><span class="w"> </span><span class="kt">double</span><span class="w"> </span><span class="n">_price</span><span class="p">;</span>

<span class="w">    </span><span class="k">public</span><span class="w"> </span><span class="kt">string</span><span class="w"> </span><span class="nf">GetName</span><span class="p">()</span>
<span class="w">    </span><span class="p">{</span>
<span class="w">        </span><span class="k">return</span><span class="w"> </span><span class="s">&quot;Apple TouchPad&quot;</span><span class="p">;</span>
<span class="w">    </span><span class="p">}</span>

<span class="w">    </span><span class="k">public</span><span class="w"> </span><span class="kt">bool</span><span class="w"> </span><span class="nf">SetPrice</span><span class="p">(</span><span class="kt">double</span><span class="w"> </span><span class="n">price</span><span class="p">)</span>
<span class="w">    </span><span class="p">{</span>
<span class="w">        </span><span class="n">_price</span><span class="w"> </span><span class="o">=</span><span class="w"> </span><span class="n">price</span><span class="p">;</span>
<span class="w">        </span><span class="k">return</span><span class="w"> </span><span class="k">true</span><span class="p">;</span>
<span class="w">    </span><span class="p">}</span>
<span class="p">}</span>

<span class="cm">/* Almost same as Factory, just an additional exposure to do something with the created method */</span>
<span class="k">public</span><span class="w"> </span><span class="k">abstract</span><span class="w"> </span><span class="k">class</span><span class="w"> </span><span class="nc">ProductAbstractFactory</span>

<span class="w">    </span><span class="k">protected</span><span class="w"> </span><span class="k">abstract</span><span class="w"> </span><span class="n">IProduct</span><span class="w"> </span><span class="nf">MakeProduct</span><span class="p">();</span>

<span class="w">    </span><span class="p">{</span>

<span class="p">}</span>

<span class="k">public</span><span class="w"> </span><span class="k">class</span><span class="w"> </span><span class="nc">PhoneConcreteFactory</span><span class="w"> </span><span class="p">:</span><span class="w"> </span><span class="n">ProductAbstractFactory</span>
<span class="p">{</span>
<span class="w">    </span><span class="k">protected</span><span class="w"> </span><span class="k">override</span><span class="w"> </span><span class="n">IProduct</span><span class="w"> </span><span class="nf">MakeProduct</span><span class="p">()</span>

<p>In this example, <code>MakeProduct</code> is used in <code>concreteFactory</code>. As a result, <code>MakeProduct()</code> may be invoked in order to retrieve it from the <code>IProduct</code>. Custom logic could run after the object is obtained in the concrete factory method. <code>GetObject</code> is made abstract in the factory interface.
</p>
<div class="mw-heading mw-heading4"><h4 id="Java"><a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a></h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=9" title="Edit section: Java"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>This <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> example is similar to one in the book <i><a href="/wiki/Design_Patterns" title="Design Patterns">Design Patterns</a>.</i>
</p><p><span class="mw-default-size" typeof="mw:File"><a href="/wiki/File:Maze_game_UML.svg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/f/f8/Maze_game_UML.svg/1005px-Maze_game_UML.svg.png" decoding="async" width="1005" height="235" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/f8/Maze_game_UML.svg/1508px-Maze_game_UML.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/f/f8/Maze_game_UML.svg/2010px-Maze_game_UML.svg.png 2x" data-file-width="1005" data-file-height="235" /></a></span>
</p><p>The <code>MazeGame</code> uses <code>Room</code> but delegates the responsibility of creating <code>Room</code> objects to its subclasses that create the concrete classes. The regular game mode could use this template method:
</p>
<div class="mw-highlight mw-highlight-lang-java mw-content-ltr" dir="ltr"><pre><span></span><span class="kd">public</span><span class="w"> </span><span class="kd">abstract</span><span class="w"> </span><span class="kd">class</span> <span class="nc">Room</span><span class="w"> </span><span class="p">{</span>

<span class="p">}</span>

<span class="w">    </span><span class="kd">public</span><span class="w"> </span><span class="kt">void</span><span class="w"> </span><span class="nf">connect</span><span class="p">(</span><span class="n">Room</span><span class="w"> </span><span class="n">room</span><span class="p">)</span><span class="w"> </span><span class="p">{}</span>

<span class="w">          </span><span class="n">Room</span><span class="w"> </span><span class="n">room2</span><span class="w"> </span><span class="o">=</span><span class="w"> </span><span class="n">makeRoom</span><span class="p">();</span>

<span class="w">          </span><span class="n">rooms</span><span class="p">.</span><span class="na">add</span><span class="p">(</span><span class="n">room2</span><span class="p">);</span>

<span class="w">     </span><span class="kd">abstract</span><span class="w"> </span><span class="kd">protected</span><span class="w"> </span><span class="n">Room</span><span class="w"> </span><span class="nf">makeRoom</span><span class="p">();</span>

It refers to the <code>makeRoom()</code> factory method that encapsulates the creation of rooms such that other rooms can be used in a subclass. To implement the other game mode that has magic rooms, the <code>makeRoom</code> method may be overridden:
</p>
<div class="mw-highlight mw-highlight-lang-java mw-content-ltr" dir="ltr"><pre><span></span><span class="kd">public</span><span class="w"> </span><span class="kd">class</span> <span class="nc">MagicMazeGame</span><span class="w"> </span><span class="kd">extends</span><span class="w"> </span><span class="n">MazeGame</span><span class="w"> </span><span class="p">{</span>

<span class="p">}</span>

<span class="w">    </span><span class="nd">@Override</span>
<span class="w">    </span><span class="kd">protected</span><span class="w"> </span><span class="n">OrdinaryRoom</span><span class="w"> </span><span class="nf">makeRoom</span><span class="p">()</span><span class="w"> </span><span class="p">{</span>
<span class="w">        </span><span class="k">return</span><span class="w"> </span><span class="k">new</span><span class="w"> </span><span class="n">OrdinaryRoom</span><span class="p">();</span>
<span class="w">    </span><span class="p">}</span>
<span class="p">}</span>

</pre></div>
<div class="mw-heading mw-heading4"><h4 id="PHP"><a href="/wiki/PHP" title="PHP">PHP</a></h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=10" title="Edit section: PHP"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>This <a href="/wiki/PHP" title="PHP">PHP</a> example shows interface implementations instead of subclassing (however, the same can be achieved through subclassing). The factory method can also be defined as <code>public</code>and called directly by the client code (in contrast to the previous Java example).
</p>
<div class="mw-highlight mw-highlight-lang-php mw-content-ltr" dir="ltr"><pre><span></span><span class="cm">/* Factory and car interfaces */</span>

<span class="k">interface</span> <span class="nx">CarFactory</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">makeCar</span><span class="p">()</span><span class="o">:</span> <span class="nx">Car</span><span class="p">;</span>
<span class="p">}</span>

<span class="k">interface</span> <span class="nx">Car</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">getType</span><span class="p">()</span><span class="o">:</span> <span class="nx">string</span><span class="p">;</span>
<span class="p">}</span>

<span class="cm">/* Concrete implementations of the factory and car */</span>

<span class="k">class</span> <span class="nc">SedanFactory</span> <span class="k">implements</span> <span class="nx">CarFactory</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">makeCar</span><span class="p">()</span><span class="o">:</span> <span class="nx">Car</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="k">new</span> <span class="nx">Sedan</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">class</span> <span class="nc">Sedan</span> <span class="k">implements</span> <span class="nx">Car</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">getType</span><span class="p">()</span><span class="o">:</span> <span class="nx">string</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="s1">&#39;Sedan&#39;</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="cm">/* Client */</span>

<span class="nv">$factory</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">SedanFactory</span><span class="p">();</span>
<span class="nv">$car</span> <span class="o">=</span> <span class="nv">$factory</span><span class="o">-&gt;</span><span class="na">makeCar</span><span class="p">();</span>
<span class="k">print</span> <span class="nv">$car</span><span class="o">-&gt;</span><span class="na">getType</span><span class="p">();</span>
</pre></div>
<div class="mw-heading mw-heading4"><h4 id="Python"><a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a></h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=11" title="Edit section: Python"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>This Python example employs the same as did the previous Java example.
</p>
<div class="mw-highlight mw-highlight-lang-python3 mw-content-ltr" dir="ltr"><pre><span></span><span class="kn">from</span> <span class="nn">abc</span> <span class="kn">import</span> <span class="n">ABC</span><span class="p">,</span> <span class="n">abstractmethod</span>


<span class="k">class</span> <span class="nc">MazeGame</span><span class="p">(</span><span class="n">ABC</span><span class="p">):</span>
    <span class="k">def</span> <span class="fm">__init__</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="kc">None</span><span class="p">:</span>
        <span class="bp">self</span><span class="o">.</span><span class="n">rooms</span> <span class="o">=</span> <span class="p">[]</span>
        <span class="bp">self</span><span class="o">.</span><span class="n">_prepare_rooms</span><span class="p">()</span>

    <span class="k">def</span> <span class="nf">_prepare_rooms</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="kc">None</span><span class="p">:</span>
        <span class="n">room1</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">make_room</span><span class="p">()</span>
        <span class="n">room2</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">make_room</span><span class="p">()</span>

        <span class="n">room1</span><span class="o">.</span><span class="n">connect</span><span class="p">(</span><span class="n">room2</span><span class="p">)</span>
        <span class="bp">self</span><span class="o">.</span><span class="n">rooms</span><span class="o">.</span><span class="n">append</span><span class="p">(</span><span class="n">room1</span><span class="p">)</span>
        <span class="bp">self</span><span class="o">.</span><span class="n">rooms</span><span class="o">.</span><span class="n">append</span><span class="p">(</span><span class="n">room2</span><span class="p">)</span>

    <span class="k">def</span> <span class="nf">play</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="kc">None</span><span class="p">:</span>
        <span class="nb">print</span><span class="p">(</span><span class="sa">f</span><span class="s2">&quot;Playing using </span><span class="si">{</span><span class="bp">self</span><span class="o">.</span><span class="n">rooms</span><span class="p">[</span><span class="mi">0</span><span class="p">]</span><span class="si">}</span><span class="s2">&quot;</span><span class="p">)</span>

    <span class="nd">@abstractmethod</span>
    <span class="k">def</span> <span class="nf">make_room</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
        <span class="k">raise</span> <span class="ne">NotImplementedError</span><span class="p">(</span><span class="s2">&quot;You should implement this!&quot;</span><span class="p">)</span>


<span class="k">class</span> <span class="nc">MagicMazeGame</span><span class="p">(</span><span class="n">MazeGame</span><span class="p">):</span>
    <span class="k">def</span> <span class="nf">make_room</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="s2">&quot;MagicRoom&quot;</span><span class="p">:</span>
        <span class="k">return</span> <span class="n">MagicRoom</span><span class="p">()</span>


<span class="k">class</span> <span class="nc">OrdinaryMazeGame</span><span class="p">(</span><span class="n">MazeGame</span><span class="p">):</span>
    <span class="k">def</span> <span class="nf">make_room</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="s2">&quot;OrdinaryRoom&quot;</span><span class="p">:</span>
        <span class="k">return</span> <span class="n">OrdinaryRoom</span><span class="p">()</span>


<span class="k">class</span> <span class="nc">Room</span><span class="p">(</span><span class="n">ABC</span><span class="p">):</span>
    <span class="k">def</span> <span class="fm">__init__</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="kc">None</span><span class="p">:</span>
        <span class="bp">self</span><span class="o">.</span><span class="n">connected_rooms</span> <span class="o">=</span> <span class="p">[]</span>

    <span class="k">def</span> <span class="nf">connect</span><span class="p">(</span><span class="bp">self</span><span class="p">,</span> <span class="n">room</span><span class="p">:</span> <span class="s2">&quot;Room&quot;</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="kc">None</span><span class="p">:</span>
        <span class="bp">self</span><span class="o">.</span><span class="n">connected_rooms</span><span class="o">.</span><span class="n">append</span><span class="p">(</span><span class="n">room</span><span class="p">)</span>


<span class="k">class</span> <span class="nc">MagicRoom</span><span class="p">(</span><span class="n">Room</span><span class="p">):</span>
    <span class="k">def</span> <span class="fm">__str__</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="nb">str</span><span class="p">:</span>
        <span class="k">return</span> <span class="s2">&quot;Magic room&quot;</span>


<span class="k">class</span> <span class="nc">OrdinaryRoom</span><span class="p">(</span><span class="n">Room</span><span class="p">):</span>
    <span class="k">def</span> <span class="fm">__str__</span><span class="p">(</span><span class="bp">self</span><span class="p">)</span> <span class="o">-&gt;</span> <span class="nb">str</span><span class="p">:</span>
        <span class="k">return</span> <span class="s2">&quot;Ordinary room&quot;</span>


<span class="n">ordinaryGame</span> <span class="o">=</span> <span class="n">OrdinaryMazeGame</span><span class="p">()</span>
<span class="n">ordinaryGame</span><span class="o">.</span><span class="n">play</span><span class="p">()</span>

<span class="n">magicGame</span> <span class="o">=</span> <span class="n">MagicMazeGame</span><span class="p">()</span>
<span class="n">magicGame</span><span class="o">.</span><span class="n">play</span><span class="p">()</span>

<div class="mw-heading mw-heading2"><h2 id="Uses">Uses</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=12" title="Edit section: Uses"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<ul><li>In <a href="/wiki/ADO.NET" title="ADO.NET">ADO.NET</a>, <a rel="nofollow" class="external text" href="https://docs.microsoft.com/en-us/dotnet/api/system.data.idbcommand.createparameter?view=netframework-4.8">IDbCommand.CreateParameter</a> is an example of the use of factory method to connect parallel class hierarchies.</li>
<li>In <a href="/wiki/Qt_(toolkit)" class="mw-redirect" title="Qt (toolkit)">Qt</a>, <a rel="nofollow" class="external text" href="http://qt-project.org/doc/qt-5.0/qtwidgets/qmainwindow.html#createPopupMenu">QMainWindow::createPopupMenu</a> <a rel="nofollow" class="external text" href="https://web.archive.org/web/20150719014739/http://qt-project.org/doc/qt-5.0/qtwidgets/qmainwindow.html#createPopupMenu">Archived</a> 2015-07-19 at the <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a> is a factory method declared in a framework that can be overridden in <a href="/wiki/Application_code" class="mw-redirect" title="Application code">application code</a>.</li>
<li>In <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, several factories are used in the <a rel="nofollow" class="external text" href="http://download.oracle.com/javase/1.5.0/docs/api/javax/xml/parsers/package-summary.html">javax.xml.parsers</a> package, such as javax.xml.parsers.DocumentBuilderFactory or javax.xml.parsers.SAXParserFactory.</li>
<li>In the <a href="/wiki/HTML5" title="HTML5">HTML5</a> <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> <a href="/wiki/Application_programming_interface" class="mw-redirect" title="Application programming interface">API</a>, the Document interface contains a createElement() factory method for creating specific elements of the HTMLElement interface.</li></ul>
