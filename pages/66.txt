<p><b>GraphQL</b> is a data <a href="/wiki/Query_language" title="Query language">query</a> and <a href="/wiki/Data_manipulation_language" title="Data manipulation language">manipulation</a> language that allows specifying what data is to be retrieved ("<a href="/wiki/Declarative_programming" title="Declarative programming">declarative</a> data fetching") or modified. A GraphQL server can process a client query using data from separate sources and present the results in a unified <a href="/wiki/Graph_(abstract_data_type)" title="Graph (abstract data type)">graph</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> The language is not tied to any specific <a href="/wiki/Database" title="Database">database</a> or storage engine. There are several <a href="/wiki/Open-source_software" title="Open-source software">open-source</a> <a href="/wiki/Runtime_system" title="Runtime system">runtime engines</a> for GraphQL.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><a href="/wiki/Meta_Platforms" title="Meta Platforms">Facebook</a> started GraphQL development in 2012 and released a draft specification and reference implementation as open source in 2015.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> In 2018, GraphQL was moved to the newly established GraphQL Foundation, hosted by the non-profit <a href="/wiki/Linux_Foundation" title="Linux Foundation">Linux Foundation</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup>
</p><p>On February 9, 2018, the GraphQL Schema Definition Language became part of the specification.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Many popular public APIs adopted GraphQL as the default way to access them. These include public APIs of Facebook, <a href="/wiki/GitHub" title="GitHub">GitHub</a>, <a href="/wiki/Yelp" title="Yelp">Yelp</a>, <a href="/wiki/Shopify" title="Shopify">Shopify</a> and Google Directions API.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Design">Design</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=2" title="Edit section: Design"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>GraphQL supports reading, writing (mutating), and subscribing to changes to data (realtime updates – commonly implemented using <a href="/wiki/WebSocket" title="WebSocket">WebSockets</a>).<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> A GraphQL service is created by defining types with fields, then providing functions to resolve the data for each field. The types and fields make up what is known as the <i>schema definition</i>. The functions that retrieve and map the data are called <i>resolvers</i>.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup>
</p><p>After being validated against the schema, a GraphQL query is executed by the server. The server returns a result that mirrors the shape of the original query, typically as <a href="/wiki/JSON" title="JSON">JSON</a>.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Type_system">Type system</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=3" title="Edit section: Type system"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The root type of a GraphQL schema, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">Query</code> by default, contains all of the fields that can be queried. Other types define the objects and fields that the GraphQL server can return. There are several base types, called scalars, to represent things like strings, numbers, and IDs.
</p><p>
Fields are defined as <a href="/wiki/Nullable_type" title="Nullable type">nullable</a> by default, and a trailing <a href="/wiki/Exclamation_mark" title="Exclamation mark">exclamation mark</a> can be used to make a field non-nullable (required). A field can be defined as a list by wrapping the field's type in <a href="/wiki/Bracket" title="Bracket">square brackets</a> (for example, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">authors: [String]</code>).<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup></p><div class="mw-highlight mw-highlight-lang-graphql mw-content-ltr" dir="ltr"><pre><span></span><span class="k">type</span><span class="w"> </span><span class="err">Query</span><span class="w"> </span><span class="p">{</span>
<span class="w">  </span><span class="nl">currentUser</span><span class="p">:</span><span class="w"> </span><span class="n">User</span>
<span class="p">}</span>

<span class="k">type</span><span class="w"> </span><span class="err">User</span><span class="w"> </span><span class="p">{</span>
<span class="w">  </span><span class="nl">id</span><span class="p">:</span><span class="w"> </span><span class="n">ID</span><span class="err">!</span>
<span class="w">  </span><span class="nl">name</span><span class="p">:</span><span class="w"> </span><span class="n">String</span><span class="err">!</span>
<span class="p">}</span>
</pre></div>
<div class="mw-heading mw-heading3"><h3 id="Queries">Queries</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=4" title="Edit section: Queries"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div><p>
A GraphQL query defines the exact shape of the data needed by the client.</p><div class="mw-highlight mw-highlight-lang-graphql mw-content-ltr" dir="ltr"><pre><span></span><span class="k">query</span><span class="w"> </span><span class="nf">CurrentUser</span><span class="w"> </span><span class="p">{</span>
<span class="w">  </span><span class="n">currentUser</span><span class="w"> </span><span class="p">{</span>
<span class="w">    </span><span class="n">name</span>
<span class="w">    </span><span class="n">age</span>
<span class="w">  </span><span class="p">}</span>
<span class="p">}</span>
</pre></div><p>Once validated and executed by the GraphQL server, the data is returned in the same shape.</p><div class="mw-highlight mw-highlight-lang-json mw-content-ltr" dir="ltr"><pre><span></span><span class="p">{</span>
<span class="w">  </span><span class="nt">&quot;currentUser&quot;</span><span class="p">:</span><span class="w"> </span><span class="p">{</span>
<span class="w">    </span><span class="nt">&quot;name&quot;</span><span class="p">:</span><span class="w"> </span><span class="s2">&quot;John Doe&quot;</span><span class="p">,</span>
<span class="w">    </span><span class="nt">&quot;age&quot;</span><span class="p">:</span><span class="w"> </span><span class="mi">23</span>
<span class="w">  </span><span class="p">}</span>
<span class="p">}</span>
</pre></div>
<div class="mw-heading mw-heading3"><h3 id="Mutations">Mutations</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=5" title="Edit section: Mutations"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A GraphQL mutation allows for data to be created, updated, or deleted. Mutations generally contain <i>variables</i>, which allow data to be passed into the server from the client. The mutation also defines the shape of the data that will be returned to the client after the operation is complete.
</p>
<div class="mw-highlight mw-highlight-lang-graphql mw-content-ltr" dir="ltr"><pre><span></span><span class="k">mutation</span><span class="w"> </span><span class="nf">CreateUser</span><span class="p">(</span><span class="nv">$name</span><span class="p">:</span><span class="w"> </span><span class="nb">String</span><span class="p">!,</span><span class="w"> </span><span class="nv">$age</span><span class="p">:</span><span class="w"> </span><span class="nb">Int</span><span class="p">!)</span><span class="w"> </span><span class="p">{</span>
<span class="w">  </span><span class="n">createUser</span><span class="p">(</span><span class="n">userName</span><span class="p">:</span><span class="w"> </span><span class="nv">$name</span><span class="p">,</span><span class="w"> </span><span class="n">age</span><span class="p">:</span><span class="w"> </span><span class="nv">$age</span><span class="p">)</span><span class="w"> </span><span class="p">{</span>
<span class="w">    </span><span class="n">name</span>
<span class="w">    </span><span class="n">age</span>
<span class="w">  </span><span class="p">}</span>
<span class="p">}</span>
</pre></div><p>The variables are passed as an object with fields that match the variable names in the mutation.</p><div class="mw-highlight mw-highlight-lang-json mw-content-ltr" dir="ltr"><pre><span></span><span class="p">{</span>
<span class="w">  </span><span class="nt">&quot;name&quot;</span><span class="p">:</span><span class="w"> </span><span class="s2">&quot;Han Solo&quot;</span><span class="p">,</span>
<span class="w">  </span><span class="nt">&quot;age&quot;</span><span class="p">:</span><span class="w"> </span><span class="mi">42</span>
<span class="p">}</span>
</pre></div><p>Once the operation is complete, the GraphQL server will return data matching the shape defined by the mutation.</p><div class="mw-highlight mw-highlight-lang-json mw-content-ltr" dir="ltr"><pre><span></span><span class="p">{</span>
<span class="w">  </span><span class="nt">&quot;data&quot;</span><span class="p">:</span><span class="w"> </span><span class="p">{</span>
<span class="w">    </span><span class="nt">&quot;createUser&quot;</span><span class="p">:</span><span class="w"> </span><span class="p">{</span>
<span class="w">      </span><span class="nt">&quot;name&quot;</span><span class="p">:</span><span class="w"> </span><span class="s2">&quot;Han Solo&quot;</span><span class="p">,</span>
<span class="w">      </span><span class="nt">&quot;age&quot;</span><span class="p">:</span><span class="w"> </span><span class="mi">42</span>
<span class="w">    </span><span class="p">}</span>
<span class="w">  </span><span class="p">}</span>
<span class="p">}</span>
</pre></div>
<div class="mw-heading mw-heading3"><h3 id="Subscriptions">Subscriptions</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=6" title="Edit section: Subscriptions"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div><p>
GraphQL also supports live updates sent from the server to client in an operation called a subscription. Again, the client defines the shape of the data that it needs whenever an update is made.</p><div class="mw-highlight mw-highlight-lang-graphql mw-content-ltr" dir="ltr"><pre><span></span><span class="k">subscription</span><span class="w"> </span><span class="p">{</span>
<span class="w">  </span><span class="n">newPerson</span><span class="w"> </span><span class="p">{</span>
<span class="w">    </span><span class="n">name</span>
<span class="w">    </span><span class="n">age</span>
<span class="w">  </span><span class="p">}</span>
<span class="p">}</span>
</pre></div><p>When a mutation is made through the GraphQL server that updates the associated field, data is sent to all subscribed clients in the format setup through the subscription.</p><div class="mw-highlight mw-highlight-lang-json mw-content-ltr" dir="ltr"><pre><span></span><span class="p">{</span>
<span class="w">  </span><span class="nt">&quot;newPerson&quot;</span><span class="p">:</span><span class="w"> </span><span class="p">{</span>
<span class="w">    </span><span class="nt">&quot;name&quot;</span><span class="p">:</span><span class="w"> </span><span class="s2">&quot;Jane&quot;</span><span class="p">,</span>
<span class="w">    </span><span class="nt">&quot;age&quot;</span><span class="p">:</span><span class="w"> </span><span class="mi">23</span>
<span class="w">  </span><span class="p">}</span>
<span class="p">}</span>
</pre></div>
<div class="mw-heading mw-heading3"><h3 id="Comparison_to_other_query_languages">Comparison to other query languages</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=7" title="Edit section: Comparison to other query languages"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>GraphQL does not provide a full-fledged <a href="/wiki/Graph_database" title="Graph database">graph</a> query language such as <a href="/wiki/SPARQL" title="SPARQL">SPARQL</a>, or even in <a href="/wiki/Hierarchical_and_recursive_queries_in_SQL" title="Hierarchical and recursive queries in SQL">dialects of SQL</a> that support <a href="/wiki/Transitive_closure" title="Transitive closure">transitive closure</a>.  For example, a GraphQL interface that reports the parents of an individual cannot return, in a single query, the set of all their ancestors.
</p>
<div class="mw-heading mw-heading2"><h2 id="Testing">Testing</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=8" title="Edit section: Testing"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>GraphQL APIs can be tested manually or with automated tools issuing GraphQL requests and verifying the correctness of the results. Automatic test generation is also possible.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> New requests may be produced through search-based techniques due to a typed schema and introspection capabilities.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Some of the software tools used for testing GraphQL implementations include <a href="/wiki/Postman_(software)" title="Postman (software)">Postman</a>, GraphiQL, Apollo Studio, GraphQL Hive, GraphQL Editor, and Step CI.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Practical_Implementations">Practical Implementations</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GraphQL&amp;action=edit&amp;section=9" title="Edit section: Practical Implementations"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Several practical implementations of GraphQL in full-stack applications have been documented. For instance, one guide demonstrates how GraphQL can be integrated with a MongoDB<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup> backend and a React-based frontend. The tutorial details the setup of a GraphQL server that communicates with MongoDB — including configuration of resolvers, data fetching optimizations, and error handling strategies. This example illustrates the flexibility of GraphQL in managing dynamic data interactions in modern web applications.
</p><p><br />
</p>
