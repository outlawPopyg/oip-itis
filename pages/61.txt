<p><b>Elasticsearch</b> is a <a href="/wiki/Search_engine_(computing)" title="Search engine (computing)">search engine</a> based on <a href="/wiki/Apache_Lucene" title="Apache Lucene">Apache Lucene</a>, a free and open-source search engine. It provides a distributed, <a href="/wiki/Multitenancy" title="Multitenancy">multitenant</a>-capable <a href="/wiki/Full-text_search" title="Full-text search">full-text search</a> engine with an <a href="/wiki/HTTP" title="HTTP">HTTP</a> web interface and schema-free <a href="/wiki/JSON" title="JSON">JSON</a> documents. Official clients are available in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>,<sup id="cite_ref-elasticsearch-java_2-0" class="reference"><a href="#cite_note-elasticsearch-java-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a><sup id="cite_ref-elasticsearch-net_3-0" class="reference"><a href="#cite_note-elasticsearch-net-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> (<a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>), <a href="/wiki/PHP_(programming_language)" class="mw-redirect" title="PHP (programming language)">PHP</a>,<sup id="cite_ref-elasticsearch-php_4-0" class="reference"><a href="#cite_note-elasticsearch-php-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>,<sup id="cite_ref-elasticsearch-py_5-0" class="reference"><a href="#cite_note-elasticsearch-py-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a><sup id="cite_ref-elasticsearch-ruby_6-0" class="reference"><a href="#cite_note-elasticsearch-ruby-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> and many other languages.<sup id="cite_ref-programming-language-clients_7-0" class="reference"><a href="#cite_note-programming-language-clients-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> According to the <a href="/wiki/DB-Engines_ranking" title="DB-Engines ranking">DB-Engines ranking</a>, Elasticsearch is the most popular enterprise search engine.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>

<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Elasticsearch&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Shay Banon created the precursor to Elasticsearch, called Compass, in 2004.<sup id="cite_ref-compass_9-0" class="reference"><a href="#cite_note-compass-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> While thinking about the third version of Compass he realized that it would be necessary to rewrite big parts of Compass to "create a scalable search solution".<sup id="cite_ref-compass_9-1" class="reference"><a href="#cite_note-compass-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> So he created "a solution built from the ground up to be distributed" and used a common interface, <a href="/wiki/JSON" title="JSON">JSON</a> over <a href="/wiki/HTTP" title="HTTP">HTTP</a>, suitable for programming languages other than Java as well.<sup id="cite_ref-compass_9-2" class="reference"><a href="#cite_note-compass-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> Shay Banon released the first version of Elasticsearch in February 2010.<sup id="cite_ref-initversion_10-0" class="reference"><a href="#cite_note-initversion-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p><p><a href="/wiki/Elastic_NV" title="Elastic NV">Elastic NV</a> was founded in 2012 to provide commercial services and products around Elasticsearch and related software.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> In June 2014, the company announced raising $70 million in a Series C funding round, just 18 months after forming the company. The round was led by <a href="/wiki/New_Enterprise_Associates" title="New Enterprise Associates">New Enterprise Associates</a> (NEA). Additional funders include <a href="/wiki/Benchmark_Capital" class="mw-redirect" title="Benchmark Capital">Benchmark Capital</a> and <a href="/wiki/Index_Ventures" title="Index Ventures">Index Ventures</a>. This round brought total funding to $104M.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In March 2015, the company <i>Elasticsearch</i> changed its name to <i>Elastic</i>.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In June 2018, Elastic filed for an <a href="/wiki/Initial_public_offering" title="Initial public offering">initial public offering</a> with an estimated valuation of between 1.5 and 3 billion dollars.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> On 5 October 2018, Elastic was listed on the <a href="/wiki/New_York_Stock_Exchange" title="New York Stock Exchange">New York Stock Exchange</a>.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Developed from the Found acquisition by Elastic in 2015,<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup> Elastic Cloud is a family of Elasticsearch-powered <a href="/wiki/SaaS" class="mw-redirect" title="SaaS">SaaS</a> offerings which include the Elasticsearch Service, as well as Elastic App Search Service, and Elastic Site Search Service which were developed from Elastic's acquisition of <a href="/wiki/Swiftype" title="Swiftype">Swiftype</a>.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> In late 2017, Elastic formed partnerships with <a href="/wiki/Google" title="Google">Google</a> to offer Elastic Cloud in <a href="/wiki/Google_Cloud_Platform" title="Google Cloud Platform">Google Cloud Platform (GCP)</a>, and <a href="/wiki/Alibaba_Group" title="Alibaba Group">Alibaba</a> to offer Elasticsearch and <a href="/wiki/Kibana" title="Kibana">Kibana</a> in <a href="/wiki/Alibaba_Cloud" title="Alibaba Cloud">Alibaba Cloud</a>.
</p><p>Elasticsearch Service users can create secure deployments with partners, Google Cloud Platform (GCP) and Alibaba Cloud.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Licensing_changes">Licensing changes</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Elasticsearch&amp;action=edit&amp;section=2" title="Edit section: Licensing changes"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>In January 2021, Elastic announced that starting with version 7.11, they would be relicensing their Apache 2.0 licensed code in Elasticsearch and <a href="/wiki/Kibana" title="Kibana">Kibana</a> to be dual licensed under <a href="/wiki/Server_Side_Public_License" title="Server Side Public License">Server Side Public License</a> and the Elastic License, neither of which is recognized as an <a href="/wiki/Open-source_license" title="Open-source license">open-source license</a>.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-:1_20-0" class="reference"><a href="#cite_note-:1-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> Elastic blamed <a href="/wiki/Amazon_Web_Services" title="Amazon Web Services">Amazon Web Services</a> (AWS) for this change, objecting to AWS offering Elasticsearch and Kibana <a href="/wiki/Software_as_a_service" title="Software as a service">as a service</a> directly to consumers and claiming that AWS was not appropriately collaborating with Elastic.<sup id="cite_ref-:1_20-1" class="reference"><a href="#cite_note-:1-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> Critics of the re-licensing decision predicted that it would harm Elastic's ecosystem and noted that Elastic had previously promised to "never....change the license of the Apache 2.0 code of Elasticsearch, Kibana, Beats, and Logstash". Amazon responded with plans to <a href="/wiki/Fork_(software_development)" title="Fork (software development)">fork</a> the projects and continue development under Apache License 2.0.<sup id="cite_ref-:0_22-0" class="reference"><a href="#cite_note-:0-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup> Other users of the Elasticsearch ecosystem, including <a href="/w/index.php?title=Logz.io&amp;action=edit&amp;redlink=1" class="new" title="Logz.io (page does not exist)">Logz.io</a>, <a href="/wiki/CrateDB" title="CrateDB">CrateDB</a> and <a href="/w/index.php?title=Aiven&amp;action=edit&amp;redlink=1" class="new" title="Aiven (page does not exist)">Aiven</a>, also committed to the need for a fork, leading to a discussion of how to coordinate the open source efforts.<sup id="cite_ref-24" class="reference"><a href="#cite_note-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-26" class="reference"><a href="#cite_note-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup> Due to potential trademark issues with using the name "Elasticsearch", AWS rebranded their fork as <a href="/wiki/OpenSearch_(software)" title="OpenSearch (software)">OpenSearch</a> in April 2021.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-eee_28-0" class="reference"><a href="#cite_note-eee-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In August 2024 the <a href="/wiki/GNU_Affero_General_Public_License" title="GNU Affero General Public License">GNU Affero General Public License</a> was added as an option, making Elasticsearch free and open-source once again.<sup id="cite_ref-:0_22-1" class="reference"><a href="#cite_note-:0-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Features">Features</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Elasticsearch&amp;action=edit&amp;section=3" title="Edit section: Features"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1251242444" /><table class="box-Technical plainlinks metadata ambox ambox-style ambox-technical" role="presentation"><tbody><tr><td class="mbox-image"><div class="mbox-image-div"><span typeof="mw:File"><span><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/f/f2/Edit-clear.svg/40px-Edit-clear.svg.png" decoding="async" width="40" height="40" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/f2/Edit-clear.svg/60px-Edit-clear.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/f2/Edit-clear.svg/80px-Edit-clear.svg.png 2x" data-file-width="48" data-file-height="48" /></span></span></div></td><td class="mbox-text"><div class="mbox-text-span">This article <b>may be too technical for most readers to understand</b>.<span class="hide-when-compact"> Please <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Elasticsearch&amp;action=edit">help improve it</a> to <a href="/wiki/Wikipedia:Make_technical_articles_understandable" title="Wikipedia:Make technical articles understandable">make it understandable to non-experts</a>, without removing the technical details.</span>  <span class="date-container"><i>(<span class="date">May 2023</span>)</i></span><span class="hide-when-compact"><i> (<small><a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this message</a></small>)</i></span></div></td></tr></tbody></table>
<p>Elasticsearch can be used to search any kind of document. It provides scalable search, has near <a href="/wiki/Real-time_search" class="mw-redirect" title="Real-time search">real-time search</a>, and supports <a href="/wiki/Multitenancy" title="Multitenancy">multitenancy</a>.<sup id="cite_ref-officialsite_29-0" class="reference"><a href="#cite_note-officialsite-29"><span class="cite-bracket">&#91;</span>29<span class="cite-bracket">&#93;</span></a></sup> "Elasticsearch is distributed, which means that indices can be divided into <a href="/wiki/Shard_(database_architecture)" title="Shard (database architecture)">shards</a> and each shard can have zero or more replicas. Each node hosts one or more shards and acts as a coordinator to delegate operations to the correct shard(s). Rebalancing and <a href="/wiki/Routing" title="Routing">routing</a> are done automatically".<sup id="cite_ref-officialsite_29-1" class="reference"><a href="#cite_note-officialsite-29"><span class="cite-bracket">&#91;</span>29<span class="cite-bracket">&#93;</span></a></sup> Related data is often stored in the same index, which consists of one or more primary shards, and zero or more replica shards. Once an index has been created, the number of primary shards cannot be changed.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30"><span class="cite-bracket">&#91;</span>30<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Elasticsearch is developed alongside the <a href="/wiki/Data_collection" title="Data collection">data collection</a> and <a href="/wiki/Log_file" class="mw-redirect" title="Log file">log</a>-parsing engine Logstash, the analytics and visualization platform <a href="/wiki/Kibana" title="Kibana">Kibana</a>, and the collection of lightweight data shippers called Beats. The four products are designed for use as an integrated solution, referred to as the "Elastic Stack".<sup id="cite_ref-31" class="reference"><a href="#cite_note-31"><span class="cite-bracket">&#91;</span>31<span class="cite-bracket">&#93;</span></a></sup> (Formerly the "ELK stack", short for "Elasticsearch, Logstash, Kibana".)
</p><p>Elasticsearch uses <a href="/wiki/Lucene" class="mw-redirect" title="Lucene">Lucene</a> and tries to make all its features available through the <a href="/wiki/JSON" title="JSON">JSON</a> and <a href="/wiki/Java_API" class="mw-redirect" title="Java API">Java API</a>. It supports <a href="/wiki/Faceted_search" title="Faceted search">facetting</a> and percolating (a form of <a href="/wiki/Prospective_search" title="Prospective search">prospective search</a>),<sup id="cite_ref-32" class="reference"><a href="#cite_note-32"><span class="cite-bracket">&#91;</span>32<span class="cite-bracket">&#93;</span></a></sup>&#8201;<sup id="cite_ref-33" class="reference"><a href="#cite_note-33"><span class="cite-bracket">&#91;</span>33<span class="cite-bracket">&#93;</span></a></sup> which can be useful for notifying if new documents match for registered queries. Another feature, "gateway", handles the long-term persistence of the index;<sup id="cite_ref-gateway_34-0" class="reference"><a href="#cite_note-gateway-34"><span class="cite-bracket">&#91;</span>34<span class="cite-bracket">&#93;</span></a></sup> for example, an index can be recovered from the gateway in <a href="/wiki/Event_data_recorder" title="Event data recorder">the event</a> of a server crash. Elasticsearch supports real-time <a href="/wiki/GET_requests" class="mw-redirect" title="GET requests">GET requests</a>, which makes it suitable as a <a href="/wiki/NoSQL" title="NoSQL">NoSQL</a> datastore,<sup id="cite_ref-jetslidedatabase_35-0" class="reference"><a href="#cite_note-jetslidedatabase-35"><span class="cite-bracket">&#91;</span>35<span class="cite-bracket">&#93;</span></a></sup> but it lacks <a href="/wiki/Distributed_transactions" class="mw-redirect" title="Distributed transactions">distributed transactions</a>.<sup id="cite_ref-transactions_36-0" class="reference"><a href="#cite_note-transactions-36"><span class="cite-bracket">&#91;</span>36<span class="cite-bracket">&#93;</span></a></sup>
</p><p>On 20 May 2019, Elastic made the core security features of the Elastic Stack available free of charge, including <a href="/wiki/TLS_termination_proxy" title="TLS termination proxy">TLS</a> for <a href="/wiki/Encryption" title="Encryption">encrypted</a> communications, file and native realm for creating and managing users, and role-based access control for controlling user access to cluster <a href="/wiki/API" title="API">APIs</a> and indexes.<sup id="cite_ref-37" class="reference"><a href="#cite_note-37"><span class="cite-bracket">&#91;</span>37<span class="cite-bracket">&#93;</span></a></sup> The corresponding source code is available under the “Elastic License”, a <a href="/wiki/Source-available_software" title="Source-available software">source-available</a> license.<sup id="cite_ref-38" class="reference"><a href="#cite_note-38"><span class="cite-bracket">&#91;</span>38<span class="cite-bracket">&#93;</span></a></sup> In addition, Elasticsearch now offers <a href="/wiki/SIEM" class="mw-redirect" title="SIEM">SIEM</a><sup id="cite_ref-Elastic_Blog_2019_39-0" class="reference"><a href="#cite_note-Elastic_Blog_2019-39"><span class="cite-bracket">&#91;</span>39<span class="cite-bracket">&#93;</span></a></sup> and <a href="/wiki/Machine_learning" title="Machine learning">Machine Learning</a><sup id="cite_ref-Elastic_Blog_2017_40-0" class="reference"><a href="#cite_note-Elastic_Blog_2017-40"><span class="cite-bracket">&#91;</span>40<span class="cite-bracket">&#93;</span></a></sup> as part of its offered services.
</p>
