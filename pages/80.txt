<p><b>Jenkins</b> is an <a href="/wiki/Open_source" title="Open source">open source</a> <a href="/wiki/Automation" title="Automation">automation</a> <a href="/wiki/Server_(computing)" title="Server (computing)">server</a>. It helps automate the parts of <a href="/wiki/Software_development" title="Software development">software development</a> related to <a href="/wiki/Software_build" title="Software build">building</a>, <a href="/wiki/Test_automation" title="Test automation">testing</a>, and <a href="/wiki/Software_deployment" title="Software deployment">deploying</a>, facilitating <a href="/wiki/Continuous_integration" title="Continuous integration">continuous integration</a>, and <a href="/wiki/Continuous_delivery" title="Continuous delivery">continuous delivery</a>. It is a server-based system that runs in <a href="/wiki/Java_Servlet#Container_servers" class="mw-redirect" title="Java Servlet">servlet containers</a> such as <a href="/wiki/Apache_Tomcat" title="Apache Tomcat">Apache Tomcat</a>, or by default as a stand-alone web-application in co-bundled <a href="/wiki/Jetty_(web_server)" title="Jetty (web server)">Eclipse Jetty</a>. It supports <a href="/wiki/Version_control" title="Version control">version control</a> tools, including <a href="/wiki/AccuRev_SCM" title="AccuRev SCM">AccuRev</a>, <a href="/wiki/Concurrent_Versions_System" title="Concurrent Versions System">CVS</a>, <a href="/wiki/Apache_Subversion" title="Apache Subversion">Subversion</a>, <a href="/wiki/Git" title="Git">Git</a>, <a href="/wiki/Mercurial" title="Mercurial">Mercurial</a>, <a href="/wiki/Perforce" title="Perforce">Perforce</a>, <a href="/wiki/ClearCase" class="mw-redirect" title="ClearCase">ClearCase</a>, and <a href="/wiki/Rational_Team_Concert" class="mw-redirect" title="Rational Team Concert">RTC</a>, and can execute <a href="/wiki/Apache_Ant" title="Apache Ant">Apache Ant</a>, <a href="/wiki/Apache_Maven" title="Apache Maven">Apache Maven</a>, and <a href="/wiki/Sbt_(software)" title="Sbt (software)">sbt</a> based projects as well as arbitrary <a href="/wiki/Shell_script" title="Shell script">shell scripts</a> and Windows <a href="/wiki/Batch_file" title="Batch file">batch commands</a>.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The Jenkins project was originally named <i><a href="/wiki/Hudson_(software)" title="Hudson (software)">Hudson</a></i>, and was renamed in 2011 after a dispute with <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a>, which had <a href="/wiki/Fork_(software_development)" title="Fork (software development)">forked</a> the project and claimed rights to the project name. The Oracle fork, <i>Hudson</i>, continued to be developed for a time before being donated to the <a href="/wiki/Eclipse_Foundation" title="Eclipse Foundation">Eclipse Foundation</a>. Oracle's Hudson is no longer maintained<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> and was announced as obsolete in February 2017.<sup id="cite_ref-About_Jenkins_8-0" class="reference"><a href="#cite_note-About_Jenkins-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Around 2007 Hudson became known as a better alternative to <a href="/wiki/CruiseControl" title="CruiseControl">Cruise Control</a> and other open-source build-servers.<sup id="cite_ref-dyer_1-1" class="reference"><a href="#cite_note-dyer-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-stackoverflow_9-0" class="reference"><a href="#cite_note-stackoverflow-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> At the <a href="/wiki/JavaOne" title="JavaOne">JavaOne</a> conference in May 2008 the software won the Duke's Choice Award in the Developer Solutions category.<sup id="cite_ref-javaone_10-0" class="reference"><a href="#cite_note-javaone-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p><p>During November 2010, after the acquisition of <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> by <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a>, an issue arose in the Hudson community with respect to the infrastructure used, which grew to encompass questions over the stewardship and control by Oracle.<sup id="cite_ref-hudson-missteps_11-0" class="reference"><a href="#cite_note-hudson-missteps-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> Negotiations between the principal project contributors and Oracle took place, and although there were many areas of agreement a key sticking point was the <a href="/wiki/Trademark" title="Trademark">trademarked</a> name "Hudson," after Oracle claimed the right to the name and applied for a trademark in December 2010. As a result, on January 11, 2011, a call for votes was made to change the project name from "Hudson" to "Jenkins." The proposal was overwhelmingly approved by a community vote on January 29, 2011, creating the Jenkins project.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
</p><p>On February 1, 2011, Oracle said that they intended to continue development of Hudson, and considered Jenkins a fork rather than a rename. Jenkins and Hudson therefore continued as two independent projects,<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> each claiming the other was the fork. As of June 2019, the Jenkins organization on <a href="/wiki/GitHub" title="GitHub">GitHub</a> had 667 project members and around 2,200 public repositories,<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> compared with Hudson's 28 project members and 20 public repositories with the last update in 2016.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In 2011, creator <a href="/wiki/Kohsuke_Kawaguchi" title="Kohsuke Kawaguchi">Kohsuke Kawaguchi</a> received an <a href="/wiki/O%27Reilly_Open_Source_Award" class="mw-redirect" title="O&#39;Reilly Open Source Award">O'Reilly Open Source Award</a> for his work on the Hudson/Jenkins project.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p><p>On April 20, 2016, version 2 was released with the <i>Pipeline</i> plugin enabled by default.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> The plugin allows for writing build instructions using a domain specific language based on <a href="/wiki/Apache_Groovy" title="Apache Groovy">Apache Groovy</a>.
</p><p>Jenkins replaced Hudson since February 8, 2017 in Eclipse.<sup id="cite_ref-About_Jenkins_8-1" class="reference"><a href="#cite_note-About_Jenkins-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In March 2018 <a href="/w/index.php?title=Jenkins_X&amp;action=edit&amp;redlink=1" class="new" title="Jenkins X (page does not exist)">Jenkins X</a> software project for <a href="/wiki/Kubernetes" title="Kubernetes">Kubernetes</a> was publicly presented,<sup id="cite_ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup> with support for different cloud providers including AWS EKS among others.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Builds">Builds</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=2" title="Edit section: Builds"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><a href="/wiki/Software_build" title="Software build">Builds</a> can be triggered by various means, for example:
</p>
<ul><li>a webhook that gets triggered upon pushed <a href="/wiki/Commit_(version_control)" title="Commit (version control)">commits</a> in a version control system</li>
<li>scheduling via a <a href="/wiki/Cron" title="Cron">cron</a>-like mechanism</li>
<li>requesting a specific build <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a>.</li>
<li>after the other builds in the queue have completed</li>
<li>invoked by other builds</li></ul>
<div class="mw-heading mw-heading2"><h2 id="Plugins">Plugins</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=3" title="Edit section: Plugins"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">Plugins</a> have been released for Jenkins that extend its use to projects written in languages other than <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>.  Plugins are available for integrating Jenkins with most version control systems and bug databases. Many build tools are supported via their respective plugins. Plugins can also change the way Jenkins looks or add new functionality. There are a set of plugins dedicated for the purpose of unit testing that generate test reports in various formats (for example, <a href="/wiki/JUnit" title="JUnit">JUnit</a> bundled with Jenkins, <a href="/w/index.php?title=MSTest&amp;action=edit&amp;redlink=1" class="new" title="MSTest (page does not exist)">MSTest</a>, <a href="/wiki/NUnit" title="NUnit">NUnit</a>, etc.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup>) and <a href="/wiki/Automated_testing" class="mw-redirect" title="Automated testing">automated testing</a> that supports automated tests. Builds can generate test reports in various formats supported by plugins (<a href="/wiki/JUnit" title="JUnit">JUnit</a> support is currently bundled) and Jenkins can display the reports and generate trends and render them in the GUI.
</p>
<div class="mw-heading mw-heading3"><h3 id="Mailer">Mailer</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=4" title="Edit section: Mailer"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Allows configuring email notifications for build results.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> Jenkins will send emails to the specified recipients whenever a certain important event occurs, such as:
</p>
<ol><li>Failed build.</li>
<li>Unstable build.</li>
<li>Successful build after a failed build, indicating that a crisis is over.</li>
<li>Unstable build after a successful one, indicating that there's a regression.</li></ol>
<div class="mw-heading mw-heading3"><h3 id="Credentials">Credentials</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=5" title="Edit section: Credentials"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Allows storing credentials in Jenkins. Provides a standardized API for other plugins to store and retrieve different types of credentials.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Monitoring_external_jobs">Monitoring external jobs</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=6" title="Edit section: Monitoring external jobs"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Adds the ability to monitor the result of externally executed jobs.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="SSH_agents">SSH agents</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=7" title="Edit section: SSH agents"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>This plugin allows managing agents (formerly known as slaves)<sup id="cite_ref-24" class="reference"><a href="#cite_note-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup> running on *nix machines over SSH.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup> It adds a new type of agent launch method. This launch method will
</p>
<ol><li>Open a SSH connection to the specified host as the specified username,</li>
<li>Check the default version of Java for that user,</li>
<li>[not implemented yet] If the default version is not compatible with Jenkins's <style data-mw-deduplicate="TemplateStyles:r886049734">.mw-parser-output .monospaced{font-family:monospace,monospace}</style><span class="monospaced">agent.jar</span>, try to find a proper version of Java</li>
<li>Once it has a suitable version of Java, copy the latest <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">agent.jar</span> via SFTP (falling back to scp if SFTP is not available),</li>
<li>Start the agent process.</li></ol>
<div class="mw-heading mw-heading3"><h3 id="Javadoc">Javadoc</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=8" title="Edit section: Javadoc"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>This plugin adds Javadoc support to Jenkins. This functionality used to be a part of the core, but as of Jenkins 1.431, it was split off into separate plugins.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The plugin enables the selection of "Publish Javadoc" as a post-build action, specifying the directory where the Javadoc is to be gathered and if retention is expected for each successful build.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Online_explanation">Online explanation</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=9" title="Edit section: Online explanation"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Jenkins can be used to schedule and monitor the running of a shell script via user interface instead of command prompt.
</p>
<div class="mw-heading mw-heading2"><h2 id="Security">Security</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Jenkins_(software)&amp;action=edit&amp;section=10" title="Edit section: Security"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Jenkins' security depends on two factors: access control and protection from external threats. Access control can be customized via two ways: user authentication and authorization. Protection from external threats such as <a href="/wiki/Cross-site_request_forgery" title="Cross-site request forgery">CSRF</a> attacks and malicious builds is supported as well.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup>
</p>
