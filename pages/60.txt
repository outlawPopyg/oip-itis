<p><b>Redis</b> (<span class="rt-commentedText nowrap"><span class="IPA nopopups noexcerpt" lang="en-fonipa"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/ˈ/: primary stress follows">ˈ</span><span title="&#39;r&#39; in &#39;rye&#39;">r</span><span title="/ɛ/: &#39;e&#39; in &#39;dress&#39;">ɛ</span><span title="&#39;d&#39; in &#39;dye&#39;">d</span><span title="/ɪ/: &#39;i&#39; in &#39;kit&#39;">ɪ</span><span title="&#39;s&#39; in &#39;sigh&#39;">s</span></span>/</a></span></span>;<sup id="cite_ref-RedisFAQ_7-0" class="reference"><a href="#cite_note-RedisFAQ-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> <b>Remote Dictionary Server</b>)<sup id="cite_ref-RedisFAQ_7-1" class="reference"><a href="#cite_note-RedisFAQ-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> is a <a href="/wiki/Source-available_software" title="Source-available software">source-available</a>, in-memory storage, used as a <a href="/wiki/Distributed_cache" title="Distributed cache">distributed</a>, <a href="/wiki/In-memory_database" title="In-memory database">in-memory</a> <a href="/wiki/Key%E2%80%93value_database" title="Key–value database">key–value</a> <a href="/wiki/Database" title="Database">database</a>, <a href="/wiki/Cache_(computing)" title="Cache (computing)">cache</a> and <a href="/wiki/Message_broker" title="Message broker">message broker</a>, with optional <a href="/wiki/Durability_(database_systems)" title="Durability (database systems)">durability</a>.<sup id="cite_ref-:9_9-0" class="reference"><a href="#cite_note-:9-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> Because it holds all data in <a href="/wiki/Random-access_memory" title="Random-access memory">memory</a> and because of its design, Redis offers low-<a href="/wiki/Latency_(engineering)" title="Latency (engineering)">latency</a> reads and writes, making it particularly suitable for use cases that require a cache. Redis is the most popular <a href="/wiki/NoSQL" title="NoSQL">NoSQL</a> database,<sup id="cite_ref-:0_10-0" class="reference"><a href="#cite_note-:0-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> and one of the most popular databases overall.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> Companies that use Redis include <a href="/wiki/Twitter" title="Twitter">Twitter</a>,<sup id="cite_ref-:1_14-0" class="reference"><a href="#cite_note-:1-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-:2_15-0" class="reference"><a href="#cite_note-:2-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Airbnb" title="Airbnb">Airbnb</a>,<sup id="cite_ref-:3_16-0" class="reference"><a href="#cite_note-:3-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Tinder_(app)" title="Tinder (app)">Tinder</a>,<sup id="cite_ref-:4_17-0" class="reference"><a href="#cite_note-:4-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Yahoo!_Inc._(2017%E2%80%93present)" title="Yahoo! Inc. (2017–present)">Yahoo</a>,<sup id="cite_ref-:5_18-0" class="reference"><a href="#cite_note-:5-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Adobe_Inc." title="Adobe Inc.">Adobe</a>,<sup id="cite_ref-:6_19-0" class="reference"><a href="#cite_note-:6-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Hulu" title="Hulu">Hulu</a>,<sup id="cite_ref-:7_20-0" class="reference"><a href="#cite_note-:7-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Amazon_(company)" title="Amazon (company)">Amazon</a><sup id="cite_ref-:8_21-0" class="reference"><a href="#cite_note-:8-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> and <a href="/wiki/OpenAI" title="OpenAI">OpenAI</a>.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Redis supports different kinds of <a href="/wiki/Abstract_data_structures" class="mw-redirect" title="Abstract data structures">abstract data structures</a>, such as <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a>, <a href="/wiki/List_(abstract_data_type)" title="List (abstract data type)">lists</a>, <a href="/wiki/Associative_array" title="Associative array">maps</a>, <a href="/wiki/Set_(abstract_data_type)" title="Set (abstract data type)">sets</a>, sorted sets, <a href="/wiki/HyperLogLog" title="HyperLogLog">HyperLogLogs</a>, <a href="/wiki/Bitmaps" class="mw-redirect" title="Bitmaps">bitmaps</a>, <a href="/wiki/Stream_(computing)" title="Stream (computing)">streams</a>, and <a href="/wiki/Spatial_indexes" class="mw-redirect" title="Spatial indexes">spatial indices</a>.
</p><p>The project was developed and maintained by Salvatore Sanfilippo, starting in 2009.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup> From 2015 until 2020, he led a project core team sponsored by <a href="/wiki/Redis_Labs" class="mw-redirect" title="Redis Labs">Redis Labs</a>.<sup id="cite_ref-network-world-201507_24-0" class="reference"><a href="#cite_note-network-world-201507-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup> Salvatore Sanfilippo left Redis as the maintainer in 2020.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup> In 2021 Redis Labs dropped the Labs from its name and now is known simply as "Redis".<sup id="cite_ref-26" class="reference"><a href="#cite_note-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In 2018, some modules for Redis adopted the <a href="/wiki/Server_Side_Public_License" title="Server Side Public License">SSPL</a>.<sup id="cite_ref-theregister.com_27-0" class="reference"><a href="#cite_note-theregister.com-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup> In 2024, the main Redis code switched to dual-licensed under the Redis Source Available License v2 and the <a href="/wiki/Server_Side_Public_License" title="Server Side Public License">Server Side Public License</a> v1.<sup id="cite_ref-github20240420_6-1" class="reference"><a href="#cite_note-github20240420-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p>

<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<figure class="mw-default-size mw-halign-right" typeof="mw:File/Thumb"><a href="/wiki/File:Salvatore_Sanfilippo.png" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/d/d5/Salvatore_Sanfilippo.png/220px-Salvatore_Sanfilippo.png" decoding="async" width="220" height="309" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/d/d5/Salvatore_Sanfilippo.png/330px-Salvatore_Sanfilippo.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/d/d5/Salvatore_Sanfilippo.png/440px-Salvatore_Sanfilippo.png 2x" data-file-width="544" data-file-height="763" /></a><figcaption>Salvatore Sanfilippo, the original developer of Redis (photo taken in 2015)</figcaption></figure>
<p>The name <i>Redis</i> means Remote Dictionary Server.<sup id="cite_ref-RedisFAQ_7-2" class="reference"><a href="#cite_note-RedisFAQ-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> The Redis project began when Salvatore Sanfilippo, nicknamed <i>antirez</i>, the original developer of Redis, was trying to improve the scalability of his Italian startup, developing a <a href="/wiki/Real-time_computing" title="Real-time computing">real-time</a> web log analyzer. After encountering significant problems in scaling some types of workloads using traditional database systems, Sanfilippo began in 2009 to prototype a first proof of concept version of Redis in <a href="/wiki/Tcl" title="Tcl">Tcl</a>.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup> Later Sanfilippo translated that prototype to the C language and implemented the first data type, the list. After a few weeks of using the project internally with success, Sanfilippo decided to open source it, announcing the project on <a href="/wiki/Hacker_News" title="Hacker News">Hacker News</a>. The project began to get traction, particularly among the Ruby community, with <a href="/wiki/GitHub" title="GitHub">GitHub</a> and <a href="/wiki/Instagram" title="Instagram">Instagram</a> being among the first companies adopting it.<sup id="cite_ref-29" class="reference"><a href="#cite_note-29"><span class="cite-bracket">&#91;</span>29<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-30" class="reference"><a href="#cite_note-30"><span class="cite-bracket">&#91;</span>30<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Sanfilippo was hired by <a href="/wiki/VMware" title="VMware">VMware</a> in March 2010.<sup id="cite_ref-31" class="reference"><a href="#cite_note-31"><span class="cite-bracket">&#91;</span>31<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-32" class="reference"><a href="#cite_note-32"><span class="cite-bracket">&#91;</span>32<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-33" class="reference"><a href="#cite_note-33"><span class="cite-bracket">&#91;</span>33<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In May 2013, Redis was sponsored by <a href="/wiki/Pivotal_Software" title="Pivotal Software">Pivotal Software</a> (a VMware spin-off).<sup id="cite_ref-34" class="reference"><a href="#cite_note-34"><span class="cite-bracket">&#91;</span>34<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In June 2015, development became sponsored by <a href="/wiki/Redis_Labs" class="mw-redirect" title="Redis Labs">Redis Labs</a>.<sup id="cite_ref-35" class="reference"><a href="#cite_note-35"><span class="cite-bracket">&#91;</span>35<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In August 2018, Redis Labs announced the switch to a <a href="/wiki/Source-available_software" title="Source-available software">source-available software</a> license, <a href="/wiki/Server_Side_Public_License" title="Server Side Public License">Server Side Public License</a> for some of the optional modules available for Redis,<sup id="cite_ref-theregister.com_27-1" class="reference"><a href="#cite_note-theregister.com-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup> while affirming that the core software will remain BSD-licensed.<sup id="cite_ref-36" class="reference"><a href="#cite_note-36"><span class="cite-bracket">&#91;</span>36<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In October 2018, Redis 5.0 was released, introducing Redis Stream – a new data structure that allows storage of multiple fields and string values with an automatic, time-based sequence at a single key.<sup id="cite_ref-redis.com_37-0" class="reference"><a href="#cite_note-redis.com-37"><span class="cite-bracket">&#91;</span>37<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In June 2020, Salvatore Sanfilippo stepped down as Redis' sole maintainer. Sanfilippo was succeeded by Yossi Gottlieb and Oran Agra.<sup id="cite_ref-38" class="reference"><a href="#cite_note-38"><span class="cite-bracket">&#91;</span>38<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-39" class="reference"><a href="#cite_note-39"><span class="cite-bracket">&#91;</span>39<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In March 2024, Redis switched to a different proprietary software license, styled as the Redis Source Available License v2, and the <a href="/wiki/Server_Side_Public_License" title="Server Side Public License">Server Side Public License</a> v1, also for its core repository,<sup id="cite_ref-github20240420_6-2" class="reference"><a href="#cite_note-github20240420-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> breaking the 2018 pledge. As a consequence, the <a href="/wiki/Linux_Foundation" title="Linux Foundation">Linux Foundation</a> created a <a href="/wiki/Fork_(source_control)" class="mw-redirect" title="Fork (source control)">fork</a> under the name of <a href="/wiki/Valkey" title="Valkey">Valkey</a>,<sup id="cite_ref-40" class="reference"><a href="#cite_note-40"><span class="cite-bracket">&#91;</span>40<span class="cite-bracket">&#93;</span></a></sup> allowing community maintainers, contributors, and users to continue working on an open source version of the Redis database.
</p>
<div class="mw-heading mw-heading2"><h2 id="Differences_from_other_database_systems">Differences from other database systems</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=2" title="Edit section: Differences from other database systems"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Redis popularized the idea of a system that can be considered a store and a <a href="/wiki/Cache_(computing)" title="Cache (computing)">cache</a> at the same time. It was designed so that data is always modified and read from the main computer memory, but also stored on disk in a format that is unsuitable for random data access. The formatted data is only reconstructed into memory once the system restarts.
</p><p>Redis also provides a data model that is very unusual compared to a <a href="/wiki/Relational_database_management_system" class="mw-redirect" title="Relational database management system">relational database management system</a> (RDBMS). User commands do not describe a query to be executed by the database engine but rather specific operations that are performed on given abstract data types. Therefore data must be stored in a way which is suitable later for fast retrieval. The retrieval is done without help from the database system in form of secondary indexes, aggregations or other common features of traditional RDBMS. The Redis implementation makes heavy use of the <a href="/wiki/Fork_(system_call)" title="Fork (system call)">fork</a> system call, to duplicate the process holding the data, so that the parent process continues to serve clients while the child process writes the in-memory data to disk.
</p>

<div class="mw-heading mw-heading2"><h2 id="Data_types">Data types</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=5" title="Edit section: Data types"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Redis maps keys to types of values. An important difference between Redis and other <a href="/wiki/Structured_storage" class="mw-redirect" title="Structured storage">structured storage</a> systems is that Redis supports not only <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a>, but also abstract data types:
</p>
<ul><li><a href="/wiki/List_(computing)" class="mw-redirect" title="List (computing)">Lists</a> of strings</li>
<li><a href="/wiki/Set_(abstract_data_type)" title="Set (abstract data type)">Sets</a> of strings (collections of non-repeating unsorted elements)</li>
<li>Sorted sets of strings (collections of non-repeating elements ordered by a <a href="/wiki/Floating-point_number" class="mw-redirect" title="Floating-point number">floating-point number</a> called score)</li>
<li><a href="/wiki/Hash_table" title="Hash table">Hash tables</a> where keys and values are strings</li>
<li><a href="/wiki/HyperLogLog" title="HyperLogLog">HyperLogLogs</a> used for approximated set cardinality size estimation, available since Redis 2.8.9 in April 2014.<sup id="cite_ref-redis.com_37-1" class="reference"><a href="#cite_note-redis.com-37"><span class="cite-bracket">&#91;</span>37<span class="cite-bracket">&#93;</span></a></sup></li>
<li><a href="/wiki/Stream_(computing)" title="Stream (computing)">Stream</a> of entries with consumer groups, allows you to store multiple fields and string values with an automatic, time-based sequence at a single key, available since Redis 5.0 in October 2018<sup id="cite_ref-redis.com_37-2" class="reference"><a href="#cite_note-redis.com-37"><span class="cite-bracket">&#91;</span>37<span class="cite-bracket">&#93;</span></a></sup></li>
<li>Geospatial data through the implementation of the <a href="/wiki/Geohash" title="Geohash">geohash</a> technique, available since Redis 3.2.<sup id="cite_ref-geo_49-0" class="reference"><a href="#cite_note-geo-49"><span class="cite-bracket">&#91;</span>49<span class="cite-bracket">&#93;</span></a></sup></li></ul>
<p>The type of a value determines what operations (called commands) are available for the value. Redis supports high-level, atomic, server-side operations like intersection, union, and difference between sets and sorting of lists, sets and sorted sets.

<li><a href="/wiki/Bloom_filter" title="Bloom filter">Bloom filter</a>, <a href="/wiki/Cuckoo_filter" title="Cuckoo filter">Cuckoo filter</a>, <a href="/wiki/Count%E2%80%93min_sketch" title="Count–min sketch">Count–min sketch</a>, and Top-K – RedisBloom<sup id="cite_ref-55" class="reference"><a href="#cite_note-55"><span class="cite-bracket">&#91;</span>55<span class="cite-bracket">&#93;</span></a></sup> implements a set of probabilistic data structures for Redis</li>
<li>Others<sup id="cite_ref-56" class="reference"><a href="#cite_note-56"><span class="cite-bracket">&#91;</span>56<span class="cite-bracket">&#93;</span></a></sup></li></ul>

<div class="mw-heading mw-heading2"><h2 id="Persistence">Persistence</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=6" title="Edit section: Persistence"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Redis typically holds the whole dataset in memory. Versions up to 2.4 could be configured to use what they refer to as virtual memory<sup id="cite_ref-60" class="reference"><a href="#cite_note-60"><span class="cite-bracket">&#91;</span>60<span class="cite-bracket">&#93;</span></a></sup> in which some of the dataset is stored on disk, but this feature is deprecated. <a href="/wiki/Persistence_(computer_science)" title="Persistence (computer science)">Persistence</a> in Redis can be achieved through two different methods. First by snapshotting, where the dataset is asynchronously transferred from memory to disk at regular intervals as a binary dump, using the Redis RDB Dump File Format. Alternatively by <a href="/wiki/Transaction_log" title="Transaction log">journaling</a>, where a record of each operation that modifies the dataset is added to an <a href="/wiki/Append" title="Append">append</a>-only file (AOF) in a background process. Redis can rewrite the append-only file in the background to avoid an indefinite growth of the journal. Journaling was introduced in version 1.1 and is generally considered the safer approach.
</p><p>By default, Redis writes data to a file system at least every 2 seconds, with more or less robust options available if needed. In the case of a complete system failure on default settings, only a few seconds of data would be lost.
</p>
<div class="mw-heading mw-heading2"><h2 id="Replication">Replication</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=7" title="Edit section: Replication"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Redis supports <a href="/wiki/Replication_(computing)" title="Replication (computing)">master–replica replication</a>. Data from any Redis server can replicate to any number of replicas. A replica may be a master to another replica. This allows Redis to implement a single-rooted replication tree. Redis replicas can be configured to accept writes, permitting intentional and unintentional inconsistency between instances. The <a href="/wiki/Publish%E2%80%93subscribe_pattern" title="Publish–subscribe pattern">publish–subscribe</a> feature is fully implemented, so a client of a replica may subscribe to a channel and receive a full feed of messages published to the master, anywhere up the replication tree. Replication is useful for read (but not write) scalability or data redundancy.<sup id="cite_ref-61" class="reference"><a href="#cite_note-61"><span class="cite-bracket">&#91;</span>61<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Performance">Performance</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=8" title="Edit section: Performance"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>When the <a href="/wiki/Durability_(database_systems)" title="Durability (database systems)">durability</a> of data is not needed, the in-memory nature of Redis allows it to perform well compared to database systems that write every change to disk before considering a transaction committed.<sup id="cite_ref-RedisFAQ_7-3" class="reference"><a href="#cite_note-RedisFAQ-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> Redis operates as a single process and is single-threaded or double-threaded when it rewrites the AOF (append-only file).<sup id="cite_ref-62" class="reference"><a href="#cite_note-62"><span class="cite-bracket">&#91;</span>62<span class="cite-bracket">&#93;</span></a></sup> Thus, a single Redis instance cannot use parallel execution of tasks such as <a href="/wiki/Stored_procedure" title="Stored procedure">stored procedures</a>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Clustering">Clustering</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=9" title="Edit section: Clustering"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Redis introduced clustering in April 2015 with the release of version 3.0.<sup id="cite_ref-cluster-release_63-0" class="reference"><a href="#cite_note-cluster-release-63"><span class="cite-bracket">&#91;</span>63<span class="cite-bracket">&#93;</span></a></sup> The <a href="/wiki/Computer_cluster" title="Computer cluster">cluster</a> specification implements a subset of Redis commands: all single-key commands are available, multi-key operations (commands related to unions and intersections) are restricted to keys belonging to the same node, and commands related to database selection operations are unavailable.<sup id="cite_ref-clustercommands_64-0" class="reference"><a href="#cite_note-clustercommands-64"><span class="cite-bracket">&#91;</span>64<span class="cite-bracket">&#93;</span></a></sup> A Redis cluster can scale up to 1,000 nodes, achieve "acceptable" write safety and to continue operations when some nodes fail.<sup id="cite_ref-cluster-commands_65-0" class="reference"><a href="#cite_note-cluster-commands-65"><span class="cite-bracket">&#91;</span>65<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-cluster-partitions_66-0" class="reference"><a href="#cite_note-cluster-partitions-66"><span class="cite-bracket">&#91;</span>66<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Use_cases">Use cases</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Redis&amp;action=edit&amp;section=10" title="Edit section: Use cases"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Due to the nature of the database design, typical use cases are session caching, full page cache, message queue applications, leaderboards and counting among others.<sup id="cite_ref-objectrocket_67-0" class="reference"><a href="#cite_note-objectrocket-67"><span class="cite-bracket">&#91;</span>67<span class="cite-bracket">&#93;</span></a></sup> The publish–subscribe messaging paradigm allows real-time communication between servers.
</p><p><a href="/wiki/Amazon_Web_Services" title="Amazon Web Services">Amazon Web Services</a> offers a managed Redis service called <a href="/wiki/Amazon_ElastiCache" title="Amazon ElastiCache">ElastiCache</a> for Redis, <a href="/wiki/Google_Cloud_Platform" title="Google Cloud Platform">Google</a> offers a managed Redis service called Cloud Memorystore,<sup id="cite_ref-68" class="reference"><a href="#cite_note-68"><span class="cite-bracket">&#91;</span>68<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> offers Azure Cache for Redis in <a href="/wiki/Microsoft_Azure" title="Microsoft Azure">Azure</a>,<sup id="cite_ref-Redis_Azure_69-0" class="reference"><a href="#cite_note-Redis_Azure-69"><span class="cite-bracket">&#91;</span>69<span class="cite-bracket">&#93;</span></a></sup> and <a href="/wiki/Alibaba_Group" title="Alibaba Group">Alibaba</a> offers ApsaraDB for Redis in <a href="/wiki/Alibaba_Cloud" title="Alibaba Cloud">Alibaba Cloud</a>.<sup id="cite_ref-Redis_AlibabaCloud_70-0" class="reference"><a href="#cite_note-Redis_AlibabaCloud-70"><span class="cite-bracket">&#91;</span>70<span class="cite-bracket">&#93;</span></a></sup>
