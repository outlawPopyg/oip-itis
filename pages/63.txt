<p><b>RabbitMQ</b> is an open-source <a href="/wiki/Message_broker" title="Message broker">message-broker</a> software (sometimes called <a href="/wiki/Message-oriented_middleware" title="Message-oriented middleware">message-oriented middleware</a>) that originally implemented the <a href="/wiki/Advanced_Message_Queuing_Protocol" title="Advanced Message Queuing Protocol">Advanced Message Queuing Protocol</a> (AMQP) and has since been extended with a <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug-in architecture</a> to support <a href="/wiki/Streaming_Text_Oriented_Messaging_Protocol" title="Streaming Text Oriented Messaging Protocol">Streaming Text Oriented Messaging Protocol</a> (STOMP), <a href="/wiki/MQ_Telemetry_Transport" class="mw-redirect" title="MQ Telemetry Transport">MQ Telemetry Transport</a> (MQTT), and other protocols.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Written in <a href="/wiki/Erlang_(programming_language)" title="Erlang (programming language)">Erlang</a>, the RabbitMQ server is built on the <a href="/wiki/Open_Telecom_Platform" title="Open Telecom Platform">Open Telecom Platform</a> framework for clustering and failover. Client libraries to interface with the broker are available for all major programming languages. The source code is released under the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">Mozilla Public License</a>.
</p><p>Since November 2020, there are commercial offerings available of RabbitMQ, for support and enterprise features: "VMware RabbitMQ OVA", "VMware RabbitMQ" and "VMware RabbitMQ for Kubernetes" (different feature levels) <sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> Open-Source RabbitMQ is also packaged by <a href="/wiki/Bitnami" title="Bitnami">Bitnami</a><sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> and commercially for VMware's Tanzu Application Service.
</p>

<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RabbitMQ&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Originally developed by Rabbit Technologies Ltd. which started as a joint venture between LShift and CohesiveFT in 2007,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> RabbitMQ was acquired in April 2010 by <a href="/wiki/SpringSource" class="mw-redirect" title="SpringSource">SpringSource</a>, a division of <a href="/wiki/VMware" title="VMware">VMware</a>.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> The project became part of <a href="/wiki/Pivotal_Software" title="Pivotal Software">Pivotal Software</a> in May 2013.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> Which then got acquired back by VMware in December 2019.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The project consists of:
</p>
<ul><li>The RabbitMQ exchange server</li>
<li>Gateways for <a href="/wiki/AMQP" class="mw-redirect" title="AMQP">AMQP</a>, <a href="/wiki/HTTP" title="HTTP">HTTP</a>, <a href="/wiki/Streaming_Text_Oriented_Messaging_Protocol" title="Streaming Text Oriented Messaging Protocol">STOMP</a>, and <a href="/wiki/MQTT" title="MQTT">MQTT</a> protocols</li>
<li><a href="/wiki/AMQP" class="mw-redirect" title="AMQP">AMQP</a> client libraries for <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> and Erlang. (AMQP clients for other languages are available from other vendors.)</li>
<li>A plug-in platform for extensibility, with a predefined collection of supported plug-ins, including:
<ul><li>A "Shovel" plug-in that takes care of moving or copying (replicating) messages from one broker to another.</li>
<li>A "Federation" plug-in that enables efficient sharing of messages between brokers (at the exchange level).</li>
<li>A "Management" plug-in that enables monitoring and control of brokers and clusters of brokers.</li></ul></li></ul>

<div class="mw-heading mw-heading3"><h3 id="Sending">Sending</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RabbitMQ&amp;action=edit&amp;section=3" title="Edit section: Sending"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The following code fragment establishes a connection, makes sure the recipient queue exists, then sends a message and finally closes the connection.
</p>

<span class="n">connection</span> <span class="o">=</span> <span class="n">pika</span><span class="o">.</span><span class="n">BlockingConnection</span><span class="p">(</span><span class="n">pika</span><span class="o">.</span><span class="n">ConnectionParameters</span><span class="p">(</span><span class="n">host</span><span class="o">=</span><span class="s2">&quot;localhost&quot;</span><span class="p">))</span>
<span class="n">channel</span> <span class="o">=</span> <span class="n">connection</span><span class="o">.</span><span class="n">channel</span><span class="p">()</span>
<span class="n">channel</span><span class="o">.</span><span class="n">queue_declare</span><span class="p">(</span><span class="n">queue</span><span class="o">=</span><span class="s2">&quot;hello&quot;</span><span class="p">)</span>
