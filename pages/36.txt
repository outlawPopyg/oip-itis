<p><b>Angular</b> (also referred to as <b>Angular 2+</b>)<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> is a <a href="/wiki/TypeScript" title="TypeScript">TypeScript</a>-based <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">free and open-source</a> <a href="/wiki/Single-page_application" title="Single-page application">single-page</a> <a href="/wiki/Web_framework" title="Web framework">web application framework</a>. It is developed by <a href="/wiki/Google" title="Google">Google</a> and by a community of individuals and corporations. Angular is a complete rewrite from the same team that built <a href="/wiki/AngularJS" title="AngularJS">AngularJS</a>. The Angular ecosystem consists of a diverse group of over 1.7 million developers, library authors, and content creators.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> According to the Stack Overflow Developer Survey, Angular is one of the most commonly used web frameworks.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Differences_between_Angular_and_AngularJS">Differences between Angular and AngularJS</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=1" title="Edit section: Differences between Angular and AngularJS"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<figure class="mw-default-size mw-halign-right" typeof="mw:File/Thumb"><a href="/wiki/File:Architecture_of_an_Angular_2_application.png" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/d/d1/Architecture_of_an_Angular_2_application.png/330px-Architecture_of_an_Angular_2_application.png" decoding="async" width="330" height="168" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/d/d1/Architecture_of_an_Angular_2_application.png/495px-Architecture_of_an_Angular_2_application.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/d/d1/Architecture_of_an_Angular_2_application.png/660px-Architecture_of_an_Angular_2_application.png 2x" data-file-width="838" data-file-height="426" /></a><figcaption>Architecture of an Angular application, services, and dependency injection</figcaption></figure>
<p>Google designed Angular as a ground-up rewrite of AngularJS. Unlike AngularJS, Angular does not have a concept of "scope" or controllers; instead, it uses a hierarchy of components as its primary architectural characteristic.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> Angular has a different expression syntax, focusing on <code>"[ ]"</code> for <a href="/wiki/Property_(programming)" title="Property (programming)">property</a> binding, and <code>"( )"</code> for <a href="/wiki/Event_(computing)" title="Event (computing)">event</a> binding.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> Angular recommends the use of <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/TypeScript" title="TypeScript">TypeScript</a> language, which introduces features such as <a href="/wiki/Static_typing" class="mw-redirect" title="Static typing">static typing</a>, <a href="/wiki/Generic_programming" title="Generic programming">generics</a>, and <a href="/wiki/Type_annotation" class="mw-redirect" title="Type annotation">type annotations</a>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Features">Features</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=2" title="Edit section: Features"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-heading mw-heading3"><h3 id="Component-based_architecture">Component-based architecture</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=3" title="Edit section: Component-based architecture"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular uses a <a href="/wiki/Software_component" title="Software component">component</a>-based architecture, which allows developers to build encapsulated, reusable user interface elements. Each component encapsulates its own <a href="/wiki/HTML" title="HTML">HTML</a>, <a href="/wiki/CSS" title="CSS">CSS</a>, and TypeScript, making it easier to manage and test individual pieces of an application.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Data_binding">Data binding</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=4" title="Edit section: Data binding"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular supports two-way data binding,<sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup> which synchronizes data between the model and the view. This ensures that any changes in the view are automatically reflected in the model and vice versa.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Dependency_injection">Dependency injection</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=5" title="Edit section: Dependency injection"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular has a built-in <a href="/wiki/Dependency_injection" title="Dependency injection">dependency injection</a> system that makes it easier to manage and inject dependencies into components and services. This promotes modularity and easier testing.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Directives">Directives</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=6" title="Edit section: Directives"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular extends HTML with additional attributes called directives. Directives offer functionality to change the behavior or appearance of <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> elements.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Routing">Routing</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=7" title="Edit section: Routing"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular includes a router that allows developers to define and manage application states and navigation paths, making it easier to build single-page applications with complex routing.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Angular_CLI">Angular CLI</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=8" title="Edit section: Angular CLI"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The Angular CLI (<a href="/wiki/Command-line_interface" title="Command-line interface">Command Line Interface</a>) provides a set of tools for creating, building, testing, and deploying Angular applications. It enables rapid application setup and simplifies ongoing development tasks.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Server-side_rendering">Server-side rendering</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=9" title="Edit section: Server-side rendering"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular has official support for server-side rendering, which improves an application's load time and performance. Server-side rendering also enhances search engine optimization by making content more accessible to web crawlers.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=10" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Angular 2.0 was announced at the ng-Europe conference 22–23 October 2014.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> On April 30, 2015, the Angular developers announced that Angular 2 moved from Alpha to Developer Preview.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup> Angular 2 moved to Beta in December 2015,<sup id="cite_ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup> and the first release candidate was published in May 2016.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> The final version was released on 14 September 2016.
</p><p>Version 8 of Angular introduced a new compilation and rendering pipeline, Ivy, and version 9 of Angular enabled Ivy by default. Angular 13 removed the deprecated former compiler, View Engine.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> Angular 14 introduced standalone components and Angular 17 made them the default, de-emphasizing the use of modules.
</p>
<div class="mw-heading mw-heading3"><h3 id="Naming">Naming</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=11" title="Edit section: Naming"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The rewrite of AngularJS was called "Angular 2", but this led to confusion among developers. To clarify, the team announced that separate names should be used for each framework with "<a href="/wiki/AngularJS" title="AngularJS">AngularJS</a>" referring to the 1.X versions and "Angular" without the "JS" referring to versions 2 and up.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup>
</p>

<div class="mw-heading mw-heading3"><h3 id="Future_releases">Future releases</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=13" title="Edit section: Future releases"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Since v9, the Angular team has moved all new applications to use the Ivy compiler and runtime. They will be working on Ivy to improve output bundle sizes and development speeds.<sup id="cite_ref-38" class="reference"><a href="#cite_note-38"><span class="cite-bracket">&#91;</span>38<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Each version is expected to be backward-compatible with the prior release. The Angular development team has pledged to do twice-a-year upgrades.
</p>
<div class="mw-heading mw-heading3"><h3 id="Support_policy_and_schedule">Support policy and schedule</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=14" title="Edit section: Support policy and schedule"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>All the major releases are supported for 18 months. This consists of 6 months of active support, during which regularly-scheduled updates and patches are released. </p>

<div class="mw-heading mw-heading3"><h3 id="Angular_Material">Angular Material</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=16" title="Edit section: Angular Material"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1236090951" /><div role="note" class="hatnote navigation-not-searchable">"Angular Material" redirects here. For the library of the same name for AngularJS 1.x, see <a href="/wiki/AngularJS#AngularJS_Material" title="AngularJS">AngularJS §&#160;AngularJS Material</a>.</div>
<p>Angular Material is a <a href="/wiki/User_interface" title="User interface">UI</a> component library that implements <a href="/wiki/Material_Design" title="Material Design">Material Design</a> in Angular.<sup id="cite_ref-41" class="reference"><a href="#cite_note-41"><span class="cite-bracket">&#91;</span>41<span class="cite-bracket">&#93;</span></a></sup> It provides a collection of reusable components that adhere to Google's Material Design specifications, aiming to offer a consistent user interface across different devices and platforms.
</p><p>Angular Material includes a variety of UI components such as buttons, cards, dialogs, grids, and form controls. These components are designed to be customizable and easy to integrate into Angular applications. Additional features of Angular Material include support for <a href="/wiki/Responsive_web_design" title="Responsive web design">responsive design</a>,&#160;theming, and accessibility.
</p>
<div class="mw-heading mw-heading3"><h3 id="Angular_Elements">Angular Elements</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Angular_(web_framework)&amp;action=edit&amp;section=17" title="Edit section: Angular Elements"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>In 2018, Angular 6 introduced Angular Elements, enabling developers to package Angular components as custom web elements, which are part of the <a href="/wiki/Web_components" class="mw-redirect" title="Web components">web components</a> set of web platform APIs.<sup id="cite_ref-42" class="reference"><a href="#cite_note-42"><span class="cite-bracket">&#91;</span>42<span class="cite-bracket">&#93;</span></a></sup>
</p>
