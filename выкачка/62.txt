<p><b>Apache Kafka</b> is a <a href="/wiki/Distributed_computing" title="Distributed computing">distributed</a> <a href="/wiki/Event_store" title="Event store">event store</a> and <a href="/wiki/Stream_processing" title="Stream processing">stream-processing</a> platform. It is an <a href="/wiki/Open-source_software" title="Open-source software">open-source</a> system developed by the <a href="/wiki/Apache_Software_Foundation" class="mw-redirect" title="Apache Software Foundation">Apache Software Foundation</a> written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> and <a href="/wiki/Scala_(programming_language)" title="Scala (programming language)">Scala</a>. The project aims to provide a unified, high-throughput, low-latency platform for handling real-time data feeds. Kafka can connect to external systems (for data import/export) via Kafka Connect, and provides the Kafka Streams <a href="/wiki/Library_(computing)" title="Library (computing)">libraries</a> for stream processing applications. Kafka uses a binary <a href="/wiki/Transmission_Control_Protocol" title="Transmission Control Protocol">TCP</a>-based protocol that is optimized for efficiency and relies on a "message set" abstraction that naturally groups messages together to reduce the overhead of the network roundtrip. This "leads to larger network packets, larger sequential disk operations, contiguous memory blocks [...] which allows Kafka to turn a bursty stream of random message writes into linear writes."<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p>

<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Kafka&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Kafka was originally developed at <a href="/wiki/LinkedIn" title="LinkedIn">LinkedIn</a>, and was subsequently open sourced in early 2011. <a href="/w/index.php?title=Jay_Kreps&amp;action=edit&amp;redlink=1" class="new" title="Jay Kreps (page does not exist)">Jay Kreps</a>, <a href="/wiki/Neha_Narkhede" title="Neha Narkhede">Neha Narkhede</a> and <a href="/w/index.php?title=Jun_Rao&amp;action=edit&amp;redlink=1" class="new" title="Jun Rao (page does not exist)">Jun Rao</a> helped co-create Kafka.<sup id="cite_ref-ForbesKreps_5-0" class="reference"><a href="#cite_note-ForbesKreps-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> Graduation from the Apache Incubator occurred on 23 October 2012.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>  Jay Kreps chose to name the software after the author <a href="/wiki/Franz_Kafka" title="Franz Kafka">Franz Kafka</a> because it is "a system optimized for writing", and he liked Kafka's work.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Applications">Applications</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Kafka&amp;action=edit&amp;section=2" title="Edit section: Applications"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Apache Kafka is based on the <a href="/wiki/Commit_log" class="mw-redirect" title="Commit log">commit log</a>, and it allows users to subscribe to it and publish data to any number of systems or real-time applications. Example applications include managing passenger and driver matching at <a href="/wiki/Uber" title="Uber">Uber</a>, providing real-time analytics and <a href="/wiki/Predictive_maintenance" title="Predictive maintenance">predictive maintenance</a> for <a href="/wiki/British_Gas" title="British Gas">British Gas</a> smart home, and performing numerous real-time services across all of LinkedIn.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
</p>

<figure class="mw-default-size mw-halign-left" typeof="mw:File/Thumb"><a href="/wiki/File:Overview_of_Apache_Kafka.svg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/6/64/Overview_of_Apache_Kafka.svg/220px-Overview_of_Apache_Kafka.svg.png" decoding="async" width="220" height="167" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/6/64/Overview_of_Apache_Kafka.svg/330px-Overview_of_Apache_Kafka.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/6/64/Overview_of_Apache_Kafka.svg/440px-Overview_of_Apache_Kafka.svg.png 2x" data-file-width="700" data-file-height="530" /></a><figcaption>Overview of Kafka</figcaption></figure>
<p>Kafka stores key-value messages that come from arbitrarily many processes called <i>producers</i>. The data can be partitioned into different "partitions" within different "topics". Within a partition, messages are strictly ordered by their offsets (the position of a message within a partition), and indexed and stored together with a timestamp. Other processes called "consumers" can read messages from partitions. For stream processing, Kafka offers the Streams API that allows writing Java applications that consume data from Kafka and write results back to Kafka. Apache Kafka also works with external stream processing systems such as <a href="/wiki/Apache_Apex" title="Apache Apex">Apache Apex</a>, <a href="/wiki/Apache_Beam" title="Apache Beam">Apache Beam</a>, <a href="/wiki/Apache_Flink" title="Apache Flink">Apache Flink</a>, <a href="/wiki/Apache_Spark" title="Apache Spark">Apache Spark</a>, <a href="/wiki/Apache_Storm" title="Apache Storm">Apache Storm</a>, and <a href="/wiki/Apache_NiFi" title="Apache NiFi">Apache NiFi</a>.
</p><p>Kafka runs on a cluster of one or more servers (called brokers), and the partitions of all topics are distributed across the cluster nodes. Additionally, partitions are replicated to multiple brokers. This architecture allows Kafka to deliver massive streams of messages in a <a href="/wiki/Fault-tolerant" class="mw-redirect" title="Fault-tolerant">fault-tolerant</a> fashion and has allowed it to replace some of the conventional messaging systems like <a href="/wiki/Java_Message_Service" class="mw-redirect" title="Java Message Service">Java Message Service</a> (JMS), <a href="/wiki/Advanced_Message_Queuing_Protocol" title="Advanced Message Queuing Protocol">Advanced Message Queuing Protocol</a> (AMQP), etc. Since the 0.11.0.0 release, Kafka offers <i>transactional writes</i>, which provide exactly-once stream processing using the Streams API.
</p><p>Kafka supports two types of topics: Regular and compacted. Regular topics can be configured with a retention time or a space bound. If there are records that are older than the specified retention time or if the space bound is exceeded for a partition, Kafka is allowed to delete old data to free storage space. By default, topics are configured with a retention time of 7 days, but it's also possible to store data indefinitely. For compacted topics, records don't expire based on time or space bounds. Instead, Kafka treats later messages as updates to earlier messages with the same key and guarantees never to delete the latest message per key. Users can delete messages entirely by writing a so-called tombstone message with null-value for a specific key.
</p><p>There are five major APIs in Kafka:
</p>
<ul><li><b>Producer API</b> – Permits an application to publish streams of records.</li>
<li><b>Consumer API</b> – Permits an application to subscribe to topics and processes streams of records.</li>
<li><b>Connect API</b> – Executes the reusable producer and consumer APIs that can link the topics to the existing applications.</li>
<li><b>Streams API</b> – This API converts the input streams to output and produces the result.</li>
<li><b>Admin API</b> – Used to manage Kafka topics, brokers, and other Kafka objects.</li></ul>
<p>The consumer and producer APIs are <a href="/wiki/Coupling_(computer_programming)" title="Coupling (computer programming)">decoupled</a> from the core functionality of Kafka through an underlying messaging <a href="/wiki/Communication_protocol" title="Communication protocol">protocol</a>. This allows writing compatible API layers in any programming language that are as efficient as the Java APIs bundled with Kafka. The Apache Kafka project maintains a list of such third party APIs.
</p>

<p>Kafka Connect (or Connect API) is a framework to import/export data from/to other systems.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> It was added in the Kafka 0.9.0.0 release and uses the Producer and Consumer API internally. The Connect framework itself executes so-called "connectors" that implement the actual logic to read/write data from other systems. The Connect API defines the programming interface that must be implemented to build a custom connector. Many open source and commercial connectors for popular data systems are available already. However, Apache Kafka itself does not include production ready connectors.
</p>
<div class="mw-heading mw-heading3"><h3 id="Streams_API">Streams API</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Kafka&amp;action=edit&amp;section=6" title="Edit section: Streams API"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Kafka Streams (or Streams API) is a stream-processing library written in Java. It was added in the Kafka 0.10.0.0 release. The library allows for the development of stateful stream-processing applications that are scalable, elastic, and fully fault-tolerant. The main API is a stream-processing <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific language</a> (DSL) that offers high-level operators like filter, <a href="/wiki/Map_(higher-order_function)" title="Map (higher-order function)">map</a>, grouping, windowing, aggregation, joins, and the notion of tables. Additionally, the Processor API can be used to implement custom operators for a more low-level development approach. The DSL and Processor API can be mixed, too. For stateful stream processing, Kafka Streams uses <a href="/wiki/RocksDB" title="RocksDB">RocksDB</a> to maintain local operator state. Because RocksDB can write to disk, the maintained state can be larger than available main memory. For fault-tolerance, all updates to local state stores are also written into a topic in the Kafka cluster. This allows recreating state by reading those topics and feed all data into RocksDB. The latest version of Streams API is 2.8.0.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup> The link also contains information about how to upgrade to the latest version.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup>
</p>

<p>Up to version 0.9.x, Kafka brokers are backward compatible with older clients only. Since Kafka 0.10.0.0, brokers are also forward compatible with newer clients. If a newer client connects to an older broker, it can only use the features the broker supports. For the Streams API, full compatibility starts with version 0.10.1.0: a 0.10.1.0 Kafka Streams application is not compatible with 0.10.0 or older brokers.
</p>
<div class="mw-heading mw-heading2"><h2 id="Performance">Performance</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Kafka&amp;action=edit&amp;section=8" title="Edit section: Performance"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Monitoring end-to-end performance requires tracking metrics from brokers, consumer, and producers, in addition to monitoring <a href="/wiki/Apache_ZooKeeper" title="Apache ZooKeeper">ZooKeeper</a>, which Kafka uses for coordination among consumers.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> There are currently several monitoring platforms to track Kafka performance. In addition to these platforms, collecting Kafka data can also be performed using tools commonly bundled with Java, including <a href="/wiki/JConsole" title="JConsole">JConsole</a>.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup>
</p>
