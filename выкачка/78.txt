<p><b>Puppet</b> is a <a href="/wiki/Software_configuration_management" title="Software configuration management">software configuration management</a> tool developed by Puppet Inc.,<sup id="cite_ref-TechCrunch_3-0" class="reference"><a href="#cite_note-TechCrunch-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> which is owned by <a href="/wiki/Perforce" title="Perforce">Perforce</a>, which is owned in turn by <a href="/wiki/Private_equity_firm" title="Private equity firm">private equity firms</a>. Puppet is used to manage stages of the <a href="/wiki/IT_infrastructure" title="IT infrastructure">IT infrastructure</a> lifecycle.<sup id="cite_ref-linuxpromagazine_4-0" class="reference"><a href="#cite_note-linuxpromagazine-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Puppet uses an <a href="/wiki/Open-core_model" title="Open-core model">open-core model</a>; its free-software version was released under version 2 of the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> (GPL) until version 2.7.0,<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> and later releases use the <a href="/wiki/Apache_License" title="Apache License">Apache License</a>, while Puppet Enterprise uses a <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> license.
</p><p>Puppet and Puppet Enterprise operate on multiple <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems (including <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/Solaris_(operating_system)" class="mw-redirect" title="Solaris (operating system)">Solaris</a>, <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">BSD</a>, <a href="/wiki/Mac_OS_X" class="mw-redirect" title="Mac OS X">Mac OS X</a>, <a href="/wiki/AIX" class="mw-redirect" title="AIX">AIX</a>, <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>) and has <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> support.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> Puppet itself is written in <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>. Facter, Puppet’s cross-platform system profiling library, is written in <a href="/wiki/C%2B%2B" title="C++">C++</a>. Puppet Server and Puppet DB are written in <a href="/wiki/Clojure" title="Clojure">Clojure</a>.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
</p>

<div class="mw-heading mw-heading2"><h2 id="Design">Design</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Puppet_(software)&amp;action=edit&amp;section=1" title="Edit section: Design"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Puppet consists of a custom <a href="/wiki/Declarative_programming" title="Declarative programming">declarative language</a> to describe system configuration. 
</p><p>Puppet is <a href="/wiki/Model-driven_architecture" title="Model-driven architecture">model-driven</a>, requiring limited programming knowledge to use.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Puppet is designed to manage the configuration of <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> systems <a href="/wiki/Declarative_programming" title="Declarative programming">declaratively</a>. 
</p>
<div class="mw-heading mw-heading3"><h3 id="Architecture">Architecture</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Puppet_(software)&amp;action=edit&amp;section=2" title="Edit section: Architecture"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<figure class="mw-default-size mw-halign-right" typeof="mw:File/Thumb"><a href="/wiki/File:140228puppetrunExampleManuallyInvokedPackageUpdate.png" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/3/34/140228puppetrunExampleManuallyInvokedPackageUpdate.png/220px-140228puppetrunExampleManuallyInvokedPackageUpdate.png" decoding="async" width="220" height="154" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/34/140228puppetrunExampleManuallyInvokedPackageUpdate.png/330px-140228puppetrunExampleManuallyInvokedPackageUpdate.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/3/34/140228puppetrunExampleManuallyInvokedPackageUpdate.png/440px-140228puppetrunExampleManuallyInvokedPackageUpdate.png 2x" data-file-width="800" data-file-height="560" /></a><figcaption>Puppet manually invoked on a client</figcaption></figure>
<p>Puppet follows <a href="/wiki/Client%E2%80%93server_model" title="Client–server model">client-server</a> architecture. The client is known as an agent and the server is known as the master. For testing and simple configuration, it can also be used as a stand-alone application run from the command line.
</p><p>Puppet Server is installed on one or more servers, and Puppet Agent is installed on all the machines to be managed. Puppet Agents communicate with the server and fetch configuration instructions. The Agent then applies the configuration on the system and sends a status report to the server.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-:0_11-0" class="reference"><a href="#cite_note-:0-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Puppet resource syntax:
</p>
<div class="mw-highlight mw-highlight-lang-puppet mw-content-ltr" dir="ltr"><pre><span></span><span class="na">type</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="s">&#39;title&#39;</span><span class="p">:</span>
<span class="w">  </span><span class="na">attribute</span><span class="w"> </span><span class="o">=&gt;</span><span class="w"> </span><span class="na">value</span>
<span class="p">}</span>
</pre></div>
<p>Example resource representing a <a href="/wiki/Unix" title="Unix">Unix</a> <a href="/wiki/User_(computing)" title="User (computing)">user</a>:
</p>
<div class="mw-highlight mw-highlight-lang-puppet mw-content-ltr" dir="ltr"><pre><span></span><span class="k">user</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="s">&#39;harry&#39;</span><span class="p">:</span>

<p>Puppet's vendor, Puppet Inc., is a privately held <a href="/wiki/Information_technology" title="Information technology">information technology</a> (IT) automation software company based in <a href="/wiki/Portland,_Oregon" title="Portland, Oregon">Portland, Oregon</a>, <a href="/wiki/United_States" title="United States">USA</a>.
</p><p>In 2005, Puppet was founded by former CEO Luke Kanies.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2021)">citation needed</span></a></i>&#93;</sup> On Jan. 29, 2019 Yvonne Wassenaar replaced Sanjay Mirchandani as CEO. Wassenaar previously worked at <a href="/wiki/Airware" title="Airware">Airware</a>, <a href="/wiki/New_Relic" title="New Relic">New Relic</a> and <a href="/wiki/VMware" title="VMware">VMware</a>. In February 2011 Puppet released its first commercial product, Puppet Enterprise, built on its <a href="/wiki/Open-source_software" title="Open-source software">open-source</a> base, with some extra commercial components.<sup id="cite_ref-enterprisenetworkingplanet_14-0" class="reference"><a href="#cite_note-enterprisenetworkingplanet-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> Puppet purchased the infrastructure automation firm Distelli in September 2017.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup> Puppet rebranded Distelli's VM Dashboard (a <a href="/wiki/Continuous_integration" title="Continuous integration">continuous integration</a> / <a href="/wiki/Continuous_delivery" title="Continuous delivery">continuous delivery</a> product) as Puppet Pipelines for Applications,<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup> and K8s Dashboard as Puppet Pipelines for <a href="/wiki/Container_(virtualization)" class="mw-redirect" title="Container (virtualization)">Containers</a>.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> The products were made generally available in October, 2017.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup> In May 2018, Puppet released Puppet Discovery, a tool to discover and manipulate resources in hybrid networks.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup> In June 2018, Puppet raised an additional $42 million for a total of $150 million in funding. The round was led by <a href="/wiki/Cisco_Systems" class="mw-redirect" title="Cisco Systems">Cisco</a> and included <a href="/wiki/Kleiner_Perkins" title="Kleiner Perkins">Kleiner Perkins</a>, True Ventures, EDBI, and <a href="/wiki/VMware" title="VMware">VMware</a>.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> Puppet's partners include <a href="/wiki/Vmware" class="mw-redirect" title="Vmware">VMware</a>, <a href="/wiki/Amazon_Web_Services" title="Amazon Web Services">Amazon Web Services</a>, <a href="/wiki/Cisco" title="Cisco">Cisco</a>, <a href="/wiki/Openstack" class="mw-redirect" title="Openstack">OpenStack</a>, <a href="/wiki/Microsoft_Azure" title="Microsoft Azure">Microsoft Azure</a>, <a href="/wiki/Eucalyptus_(computing)" class="mw-redirect" title="Eucalyptus (computing)">Eucalyptus</a>, and Zenoss.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> 
</p><p>In April 2022, it was announced Puppet had been acquired by the <a href="/wiki/Minneapolis" title="Minneapolis">Minneapolis</a>-headquartered software developer, <a href="/wiki/Perforce" title="Perforce">Perforce</a>.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup> The company subsequently laid off 15% of Puppet's workforce in Portland.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup>
</p>
