<p><b>Progress Chef</b> (formerly <b>Chef</b>)<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> is a <a href="/wiki/Configuration_management" title="Configuration management">configuration management</a> tool written in <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> and <a href="/wiki/Erlang_(programming_language)" title="Erlang (programming language)">Erlang</a>. It uses a pure-Ruby, <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific language</a> (DSL) for writing system configuration "recipes". Chef is used to streamline the task of configuring and maintaining a company's servers, and can integrate with cloud-based platforms such as <a href="/wiki/Amazon_EC2" class="mw-redirect" title="Amazon EC2">Amazon EC2</a>, <a href="/wiki/Google_Cloud_Platform" title="Google Cloud Platform">Google Cloud Platform</a>, <a href="/wiki/Oracle_Cloud" title="Oracle Cloud">Oracle Cloud</a>, <a href="/wiki/OpenStack" title="OpenStack">OpenStack</a>, <a href="/wiki/IBM_Cloud" title="IBM Cloud">IBM Cloud</a>, <a href="/wiki/Microsoft_Azure" title="Microsoft Azure">Microsoft Azure</a>, and <a href="/wiki/Rackspace" class="mw-redirect" title="Rackspace">Rackspace</a> to automatically provision and configure new machines. Chef contains solutions for both small and large scale systems.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Features">Features</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Progress_Chef&amp;action=edit&amp;section=1" title="Edit section: Features"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The user writes "recipes" that describe how Chef manages server applications and utilities (such as <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache HTTP Server</a>, <a href="/wiki/MySQL" title="MySQL">MySQL</a>, or <a href="/wiki/Hadoop" class="mw-redirect" title="Hadoop">Hadoop</a>) and how they are to be configured. These recipes (which can be grouped together as a "cookbook" for easier management) describe a series of resources that should be in a particular state: packages that should be installed, services that should be running, or files that should be written. These various resources can be configured to specific versions of software to run and can ensure that software is installed in the correct order based on dependencies. Chef makes sure each resource is properly configured and corrects any resources that are not in the desired state.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Chef can run in <a href="/wiki/Client/server" class="mw-redirect" title="Client/server">client/server</a> mode, or in a standalone configuration named "chef-solo". In <a href="/wiki/Client/server" class="mw-redirect" title="Client/server">client/server</a> mode, the Chef client sends various attributes about the node to the Chef server. The server uses <a href="/wiki/Elasticsearch" title="Elasticsearch">Elasticsearch</a> to index these attributes and provides an API for clients to query this information. Chef recipes can query these attributes and use the resulting data to help configure the node.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (November 2018)">citation needed</span></a></i>&#93;</sup>
</p><p>Traditionally, Chef was used to manage <a href="/wiki/Linux" title="Linux">Linux</a> but later versions add support for  <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>.<sup id="cite_ref-wired_6-0" class="reference"><a href="#cite_note-wired-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>It is one of the major configuration management systems on Linux, along with <a href="/wiki/CFEngine" title="CFEngine">CFEngine</a>, <a href="/wiki/Ansible_(software)" title="Ansible (software)">Ansible</a> and <a href="/wiki/Puppet_(software)" title="Puppet (software)">Puppet</a>.<sup id="cite_ref-Chef_vs_Puppet_7-0" class="reference"><a href="#cite_note-Chef_vs_Puppet-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> More than a configuration management tool, Chef, along with Puppet and Ansible, is one of the industry's most notable <a href="/wiki/Infrastructure_as_Code" class="mw-redirect" title="Infrastructure as Code">Infrastructure as Code</a> (IAC) tools.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup>
</p>

<div class="mw-heading mw-heading3"><h3 id="Opscode">Opscode</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Progress_Chef&amp;action=edit&amp;section=3" title="Edit section: Opscode"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Chef was created by Adam Jacob as a tool for his consulting company, whose business model was to build end-to-end server/deployment tools. Jacob showed Chef to <a href="/wiki/Jesse_Robbins" title="Jesse Robbins">Jesse Robbins</a>, who saw its potential after running operations at <a href="/wiki/Amazon.com" class="mw-redirect" title="Amazon.com">Amazon</a>. They founded a new company (<b>Opscode</b>) with Barry Steinglass, Nathen Haneysmith, and Joshua Timberman to turn Chef into a product.<sup id="cite_ref-chefvideo_12-0" class="reference"><a href="#cite_note-chefvideo-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> The company Chief Technology Officer was Adam Jacob.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2021)">citation needed</span></a></i>&#93;</sup> <b>Chef Software, Inc.</b> was a corporation headquartered in <a href="/wiki/Seattle,_Washington" class="mw-redirect" title="Seattle, Washington">Seattle, Washington</a>. 
</p><p>The project was originally named "marionette", but the word was too long and cumbersome to type; naming the format modules were prepared in "recipe" led to the project being renamed "Chef".<sup id="cite_ref-chefvideo_12-1" class="reference"><a href="#cite_note-chefvideo-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In February 2013, Opscode released version 11 of Chef. Changes in this release included a complete rewrite of the core <a href="/wiki/Application_programming_interface" class="mw-redirect" title="Application programming interface">API</a> server in <a href="/wiki/Erlang_(programming_language)" title="Erlang (programming language)">Erlang</a>.<sup id="cite_ref-chef11_13-0" class="reference"><a href="#cite_note-chef11-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In Sep 2015, Chef Chef was valued at $360 million after a $40 million  venture capital funding round.<sup id="cite_ref-Venture_14-0" class="reference"><a href="#cite_note-Venture-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-Pitchbook_15-0" class="reference"><a href="#cite_note-Pitchbook-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In November 2015, the company acquired a German security startup, VulcanoSec.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In April 2019, the company announced that the source code for their software would continue to be released under the Apache 2.0 license, while binaries would only be available under the terms of a proprietary license.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> In response, the Cinc project began releasing Apache 2.0 licensed binaries of several Chef products.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In 2019, it was discovered by a journalist that <a href="/wiki/U.S._Immigration_and_Customs_Enforcement" title="U.S. Immigration and Customs Enforcement">U.S. Immigration and Customs Enforcement</a> was paying Chef approximately $95,000 per year for a software license. At that time, a former Chef employee deleted his code repository in protest of the contract. The company did not announce any changes to its contracting processes or partners.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Chef_Automate">Chef Automate</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Progress_Chef&amp;action=edit&amp;section=4" title="Edit section: Chef Automate"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Chef offered a single commercial product, Chef Automate, released at ChefConf in July 2016. Chef Automate included a full-stack continuous deployment pipeline, and automated testing for compliance and security.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> Chef Automate built on two of Chef's open source projects - Chef and InSpec - and integrated with the company's third open source project, Habitat. Habitat offered "application automation" to simplify running complex applications in different environments including containers, traditional data servers, or PaaS.<sup id="cite_ref-mw1_21-0" class="reference"><a href="#cite_note-mw1-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Chef offered three versions of its product: Chef Basics (free, open source), Hosted Chef ($72/node, minimum 20 node purchase), and Chef Automate ($137/node, annual subscription).
</p>
<div class="mw-heading mw-heading3"><h3 id="Progress">Progress</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Progress_Chef&amp;action=edit&amp;section=5" title="Edit section: Progress"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>On September 8, 2020, <a href="/wiki/Progress_Software" title="Progress Software">Progress</a> announced the acquisition of Chef,<sup id="cite_ref-24" class="reference"><a href="#cite_note-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-26" class="reference"><a href="#cite_note-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup> with an intended final date in October.
</p><p>A press release gave the price of acquisition at $220 million.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup> The merged company was named <b>Progress Chef</b>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Platform_support">Platform support</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Progress_Chef&amp;action=edit&amp;section=6" title="Edit section: Platform support"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Chef is supported on multiple platforms according to a supported platforms matrix for client and server products.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup> Major platform support for clients includes <a href="/wiki/AIX" class="mw-redirect" title="AIX">AIX</a>, <a href="/wiki/Amazon_Linux" class="mw-redirect" title="Amazon Linux">Amazon Linux</a>, <a href="/wiki/Debian" title="Debian">Debian</a>, <a href="/wiki/CentOS" title="CentOS">CentOS</a>/<a href="/wiki/RHEL" class="mw-redirect" title="RHEL">RHEL</a>, <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/MacOS" title="MacOS">macOS</a>, <a href="/wiki/Solaris_(operating_system)" class="mw-redirect" title="Solaris (operating system)">Solaris</a>, <a href="/wiki/SUSE_Linux" class="mw-redirect" title="SUSE Linux">SUSE Linux</a>, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> and <a href="/wiki/Ubuntu_(operating_system)" class="mw-redirect" title="Ubuntu (operating system)">Ubuntu</a>. Additional client platforms include <a href="/wiki/Arch_Linux" title="Arch Linux">Arch Linux</a> and <a href="/wiki/Fedora_(operating_system)" class="mw-redirect" title="Fedora (operating system)">Fedora</a>. Chef Server is supported on <a href="/wiki/Amd64" class="mw-redirect" title="Amd64">amd64</a> on <a href="/wiki/RHEL" class="mw-redirect" title="RHEL">RHEL</a>/<a href="/wiki/CentOS" title="CentOS">CentOS</a>, <a href="/wiki/Oracle_Linux" title="Oracle Linux">Oracle Linux</a>, <a href="/wiki/SUSE_Linux" class="mw-redirect" title="SUSE Linux">SUSE Linux</a> and <a href="/wiki/Ubuntu_(operating_system)" class="mw-redirect" title="Ubuntu (operating system)">Ubuntu</a>.
</p>
